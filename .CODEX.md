# Codex拡張のチャットでのファイルリンク
- Codex拡張のチャット（webview）では、生パス文字列（C:\...）は自動で青リンク化されない。
- このため「クリックでエディタを開く」必要がある場合は、Markdownリンク形式でローカルパスを指定する（例: `C:/Users/na6ce/source/repos/IndigoMovieManager/Views/Main/MainWindow.xaml.cs` をリンク先に使う）。
- リンク先はローカルパス（絶対/相対）を使うこと。Windowsでもリンク先は `C:/...` のように `/` 区切りを推奨。
- 行番号は `:120` または `#L120` 形式で指定できる。
- `https://file+.vscode-resource...` はWebView内部URLなので使わない。`cursor://` / `vscode://` / `file://` もこの環境では期待どおりに動かないため使わない。
- `file:///C:/...`（Markdownリンクのfile URI）もこの環境では安定して開けないため使わない。
