# AI向け 下部タブ目次 (2026-03-15)

## 目的

- `BottomTabs` 配下を、次に触る AI が最短で追えるようにする。
- `MainWindow` のどこを入口に見ればよいかを固定する。
- 各タブの責務境界を見失わないようにする。

## 最初に見る場所

1. `MainWindow.xaml`
   - 下部タブの host 配置を見る。
2. `MainWindow.xaml.cs`
   - 初期化線を見る。
3. `BottomTabs/Common`
   - 共通の選択制御と活性判定を見る。
4. 触る対象タブのフォルダ
   - partial と View を対で読む。

- `MainWindow.BottomTab.ThumbnailError.cs`
  - 下部タブ host へ触る最小ヘルパ。
- `MainWindow.BottomTab.ThumbnailError.Progress.cs`
  - サムネ失敗タブ専用の可視監視と 1 秒ポーリング。
  - 見えている間だけ `FailureDb` を軽く再読込し、`待機中` / `救済中` / `反映待ち` を自動追従する。
- `ThumbnailErrorTabView.xaml`
  - サムネ失敗一覧、進行状況サマリ、`リストクリア` / 救済ボタンの View。
- `ThumbnailErrorTabView.xaml.cs`
  - ボタンと一覧選択のイベント橋渡し。
- `MainWindow.ThumbnailFailedTab.cs`
  - 実際の救済投入本体。`マーカー + FailureDb 最新親行` を束ねて一覧化し、`リストクリア` は表示中の ERROR マーカーと該当 FailureDb 行を消す。
  - 一覧再読込では重い選択復元は行わず、`選択救済` は `CurrentItem` / `CurrentCell` も見て実行対象を決める。
  - `選択救済`、`一括救済`、`右クリック サムネ救済` は即時 worker 起動要求で流す。

## 現在の設計ルール

- `MainWindow.xaml` は host の殻だけを持つ。
- 各タブ固有の処理は `BottomTabs/<TabName>` へ寄せる。
- 非アクティブ時は更新せず、dirty を立てて、再表示時に 1 回だけ反映する。
- 重い MVVM 化より、追いやすい partial + UserControl を優先する。
- 上側メインタブの index を壊したくない物は、殻だけ残して実体を下部タブへ逃がしてよい。

## 変更時の見方

- 上側タブ切替で何か起きるなら `BottomTabs/Common/MainWindow.BottomTabs.Common.cs`
- 下部タブの表示条件なら対象タブの `MainWindow.BottomTab.*.cs`
- 見た目の変更なら対象タブの `*TabView.xaml`
- 高密度表示を再利用するなら `Themes/Generic.xaml` の `HighDensityBottomTab*` スタイル群
- ボタンや slider の橋渡しなら対象タブの `*TabView.xaml.cs`
