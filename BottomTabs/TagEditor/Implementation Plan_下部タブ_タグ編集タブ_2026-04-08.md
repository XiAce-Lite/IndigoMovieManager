# Implementation Plan: 下部タブ タグ編集タブ 2026-04-08

## 1. 目的
- 下部タブに新しい `タグ` タブを追加し、選択中動画のタグ確認・追加・解除を 1 か所で行えるようにする。
- 左・中・右の 3 カラムで、プレビュー、登録済みタグ、検索タグを分離して見やすくする。
- 既存のタグ更新処理、共通検索導線、詳細タブの表示資産をできるだけ再利用し、実装をシンプルに保つ。
- WhiteBrowser 互換方針を守り、`.wb` スキーマは変更しない。

## 2. 前提
- 対象は下部タブの新設であり、既存の右クリックメニューや `TagEdit` ダイアログは残す。
- 選択中動画は既存の `GetSelectedItemByTabIndex()` を正本とする。
- タグ保存は既存の `UpdateTag(...)` を再利用する。
- タグ検索は既存の共通検索導線を使う。

## 3. UI 構成

### 3.1 全体レイアウト
- 下部タブ名は `タグ`
- 横 3 分割レイアウト
- 左: 選択中動画プレビュー
- 中: カスタムタグ追加 + 登録済みタグ
- 右: 検索タグ

### 3.2 左ペイン
- 選択中動画名
- Grid 用サムネイル

### 3.3 中央ペイン
- カスタムタグ追加 textbox
- `+` ボタン
- 現在登録済みタグ一覧

### 3.4 右ペイン
- 候補タグの固定パレット
- 初期タグ
  - `★`
  - `★★`
  - `★★★`
  - `★★★★`
  - `★★★★★`

## 4. 左ペイン仕様

### 4.1 表示内容
- 選択中動画がある場合
  - 動画名を表示
  - Grid サムネイルを表示
- 選択中動画がない場合
  - プレースホルダを表示

## 5. 中央ペイン仕様

### 5.1 カスタムタグ追加
- textbox に任意タグを入力する
- `Enter` または `+` ボタンで追加する
- 改行は空白へ正規化し、1 つのタグ名として扱う
- 選択中動画がない時は textbox / `+` を無効化する

### 5.2 登録済みタグチップ
- `×`
  - 確認ダイアログは出さない
  - その動画からタグを外す
  - DB を即時更新する
- 文字 + `✓`
  - 検索条件トグルとして扱う
  - クリックで検索条件へ加算 / 減算する
  - 左タブ起点で絞り込んだ後も、同じ動画が結果内に残っていれば再選択する

## 6. 右ペイン仕様

### 5.1 基本
- 右側タグは固定表示とする
- タグ項目は消さない
- 各タグは `アクティブ / 非アクティブ` の 2 状態を持つ

### 6.2 状態ルール
- 現在の検索条件に入っているタグ
  - アクティブ色
- それ以外
  - 非アクティブ色

### 6.3 クリック時挙動
- 左の `+`
  - 選択中動画へそのタグを追加
  - DB を即時更新
- 文字 + `✓`
  - 検索条件トグル
  - アクティブなら減算、非アクティブなら加算

## 7. 操作フロー

### 6.1 動画選択変更時
- 現在の上部タブ選択から `MovieRecords` を取得
- 左ペインの動画情報を更新
- 中央ペインの登録済みタグを更新
- 右ペインの固定タグ状態を同期
- カスタムタグ textbox の有効 / 無効を同期

### 6.2 中央・右から検索
- 中央タグの `✓` または右タグの `✓` をクリック
- 現在の検索条件へタグ文字列を加減算する
- 共通検索実行へ委譲する
- 左右のチップ状態を同期する

### 6.3 タグ追加
- 中央 textbox の `Enter` または `+`
- 右タグの `+`
- どちらも `MovieRecords.Tag` / `MovieRecords.Tags` を更新
- `UpdateTag(...)` を呼ぶ
- 必要なら詳細タブも即更新する

## 8. 実装方針

### 7.1 新規追加候補
- `BottomTabs/TagEditor/TagEditorTabView.xaml`
- `BottomTabs/TagEditor/TagEditorTabView.xaml.cs`
- `BottomTabs/TagEditor/TagEditorTabPresenter.cs`
- `BottomTabs/TagEditor/MainWindow.BottomTab.TagEditor.cs`

### 7.2 再利用する既存資産
- 選択中動画取得
  - `GetSelectedItemByTabIndex()`
- タグ保存
  - `_mainDbMovieMutationFacade.UpdateTag(...)`
- 共通検索
  - `ExecuteSearchKeywordAsync(...)` 系
- 詳細表示のサムネイル資産
  - Grid サムネイル表示に使っている既存の画像/バインディング方式

### 8.3 データ表現
- 左ペイン
  - 現在の `MovieRecords` をそのまま使う
- 中央ペイン
  - `MovieRecords.Tag` を登録済みチップへ反映する
- 右ペイン
  - 固定タグ定義を持つ軽量 ViewModel または DTO を使う
  - 例
    - `TagName`
    - `IsActive`
  - `IsActive` は「現在の検索条件に入っているか」を表す

## 9. デザイン指針
- 左は「今の動画を見る場所」
- 中は「今ついているタグを編集する場所」
- 右は「検索条件を切り替える場所」
- 役割を混ぜない
- MaterialDesign の Filter Chip 寄りの見た目で、チェック状態を色とアイコンで分かるようにする
- 色だけで状態が分かるようにする
- タグチップは押しやすいサイズにする

## 10. リスクと回避策

### 9.1 既存タグ操作との二重管理
- 回避策
  - DB 更新は既存の `UpdateTag(...)` へ一本化する
  - 文字列整形も既存ルールに合わせる

### 9.2 選択変更時の表示ズレ
- 回避策
  - 詳細タブと同じ選択変更フローへぶら下げる
  - 非表示時は dirty 管理し、表示時にまとめて反映する

### 10.3 カスタムタグ入力の揺れ
- 回避策
  - 改行は空白へ置換する
  - trim して空入力を弾く
  - DB 保存前の既存タグ正規化ロジックを再利用する

## 11. 今回やらないこと
- 固定タグパレットのユーザー編集
- 階層タグ編集
- 複数動画一括編集への拡張
- `tagbar` との統合
- 右タグの D&D

- 中央の `Enter` / `+` でタグ追加できる

- 左右の `✓` で検索条件へ加算 / 減算できる
