# Implementation Plan: 下部タブ タグ編集タブ 2026-04-08

## 1. 目的
- 下部タブに新しい `タグ` タブを追加し、選択中動画のタグ確認・追加・解除を 1 か所で行えるようにする。
- 左側で「現在の実データ」、右側で「固定タグパレット」を扱い、操作意図を分かりやすく分離する。
- 既存のタグ更新処理、共通検索導線、詳細タブの表示資産をできるだけ再利用し、実装をシンプルに保つ。
- WhiteBrowser 互換方針を守り、`.wb` スキーマは変更しない。

## 2. 前提
- 対象は下部タブの新設であり、既存の右クリックメニューや `TagEdit` ダイアログは残す。
- 選択中動画は既存の `GetSelectedItemByTabIndex()` を正本とする。
- タグ保存は既存の `UpdateTag(...)` を再利用する。
- タグ検索は既存の共通検索導線を使う。

## 3. UI 構成

### 3.1 全体レイアウト
- 下部タブ名は `タグ`
- 横 2 分割レイアウト
- 左右比率は初期値 `2 : 1` を想定

### 3.2 左ペイン
- 選択中動画名
- Grid 用サムネイル
- 現在登録済みタグ一覧

### 3.3 右ペイン
- 候補タグの固定パレット
- 初期タグ
  - `★`
  - `★★`
  - `★★★`
  - `★★★★`
  - `★★★★★`

## 4. 左ペイン仕様

### 4.1 表示内容
- 選択中動画がある場合
  - 動画名を表示
  - Grid サムネイルを表示
  - 登録済みタグをチップ表示
- 選択中動画がない場合
  - プレースホルダを表示
  - タグ一覧は空

### 4.2 登録済みタグチップ
- チップ本体クリック
  - そのタグで検索する
  - 共通検索導線を通す
- `×` クリック
  - 確認ダイアログは出さない
  - その動画からタグを外す
  - DB を即時更新する
  - 左ペインからそのタグを消す
  - 右ペインは固定パレットなので、項目自体は消さず非アクティブ色へ戻す

## 5. 右ペイン仕様

### 5.1 基本
- 右側タグは固定表示とする
- タグ項目は消さない
- 各タグは `アクティブ / 非アクティブ` の 2 状態を持つ

### 5.2 状態ルール
- 選択中動画にそのタグが登録済み
  - アクティブ色
- 未登録
  - 非アクティブ色

### 5.3 クリック時挙動
- 非アクティブをクリック
  - そのタグを検索条件へ加算
  - 右ペインはアクティブ色へ切替
- アクティブをクリック
  - そのタグを検索条件から減算
  - 右ペインは非アクティブ色へ切替

### 5.4 ドラッグ時挙動
- 右ペインのタグを左ペインへドラッグ&ドロップ
  - 選択中動画へそのタグを追加
  - DB を即時更新
  - 左ペインへタグ追加
- 右ペインのタグ自体は消さない

## 6. 操作フロー

### 6.1 動画選択変更時
- 現在の上部タブ選択から `MovieRecords` を取得
- 左ペインの動画情報を更新
- 左ペインの登録済みタグを更新
- 右ペインの固定タグ状態を同期

### 6.2 左から検索
- 登録済みタグの本体クリック
- 既存の共通検索実行へ委譲
- `SearchBox` と `FilteredMovieRecs` を既存ルールで更新

### 6.3 右クリックではなくトグル検索
- 固定タグクリック
- 現在の検索条件へタグ文字列を加減算する
- 共通検索実行へ委譲する
- 右ペインのチップ状態を同期する

### 6.4 右から左へドラッグ追加
- 固定タグを左へドロップ
- `MovieRecords.Tag` / `MovieRecords.Tags` を更新
- `UpdateTag(...)` を呼ぶ
- 左ペインを同期
- 必要なら既存詳細タブも再描画

## 7. 実装方針

### 7.1 新規追加候補
- `BottomTabs/TagEditor/TagEditorTabView.xaml`
- `BottomTabs/TagEditor/TagEditorTabView.xaml.cs`
- `BottomTabs/TagEditor/TagEditorTabPresenter.cs`
- `BottomTabs/TagEditor/MainWindow.BottomTab.TagEditor.cs`

### 7.2 再利用する既存資産
- 選択中動画取得
  - `GetSelectedItemByTabIndex()`
- タグ保存
  - `_mainDbMovieMutationFacade.UpdateTag(...)`
- 共通検索
  - `ExecuteSearchKeywordAsync(...)` 系
- 詳細表示のサムネイル資産
  - Grid サムネイル表示に使っている既存の画像/バインディング方式

### 7.3 データ表現
- 左ペイン
  - 現在の `MovieRecords` をそのまま使う
- 右ペイン
  - 固定タグ定義を持つ軽量 ViewModel または DTO を使う
  - 例
    - `TagName`
    - `IsActive`
  - `IsActive` は「選択中動画の登録状態」ではなく「現在の検索条件に入っているか」を表す

## 8. デザイン指針
- 左は「今の状態を見る場所」
- 右は「検索条件を切り替える場所」
- 役割を混ぜない
- MaterialDesign の Filter Chip 寄りの見た目で、チェック状態を色とアイコンで分かるようにする
- 色だけで状態が分かるようにする
- タグチップは押しやすいサイズにする

## 9. リスクと回避策

### 9.1 既存タグ操作との二重管理
- 回避策
  - DB 更新は既存の `UpdateTag(...)` へ一本化する
  - 文字列整形も既存ルールに合わせる

### 9.2 選択変更時の表示ズレ
- 回避策
  - 詳細タブと同じ選択変更フローへぶら下げる
  - 非表示時は dirty 管理し、表示時にまとめて反映する

### 9.3 固定タグ以外との整合
- 回避策
  - このタブはまず固定タグパレットに限定する
  - 任意タグ編集は既存の `TagEdit` やタグ一覧操作を残す

## 10. 今回やらないこと
- 任意タグの新規作成 UI
- 固定タグパレットのユーザー編集
- 階層タグ編集
- 複数動画一括編集への拡張
- `tagbar` との統合

## 11. 完了条件
- 下部タブ `タグ` が追加されている
- 左側に選択中動画名、Grid サムネ、登録済みタグが表示される
- 左タグクリックで検索できる
- 左タグの `×` で即時解除できる
- 右側に `★` から `★★★★★` の固定タグパレットが表示される
- 右側チップはクリックで検索条件へ加算/減算できる
- 右側チップはドラッグして左へドロップすると選択中動画へ追加できる
- 右側の色状態が現在の検索条件と同期している

## 12. 実装順
1. `TagEditor` 下部タブの View / Presenter / MainWindow 接続を追加する
2. 左ペインへ選択中動画表示を実装する
3. 右ペインへ Filter Chip 風の固定タグパレットを実装する
4. 右クリックではなくクリック検索トグルを共通検索導線へ接続する
5. 右から左へのドラッグ追加を既存 DB 更新処理へ接続する
6. 左タグ検索と左 `×` 削除を整える
7. 選択変更時の同期と dirty 管理を整える
