# 設計メモ: メインDB読取専用接続（2026-03-15）

最終更新日: 2026-03-15

## 目的

- `*.wb` を本体が開いている間でも、不要な書き込みロックを増やしにくくする。
- 読み取りだけの処理は `ReadOnly=True` で開き、書き込み経路と役割を分ける。

## 今回の方針

1. `DB/SQLite.cs` に接続文字列生成口を追加する。
2. 読み取り専用の `GetData(...)` とスキーマ検証は `CreateReadOnlyConnection(...)` を使う。
3. rescue worker の `system.thum` 読み取りも `ReadOnly=True` で開く。
4. 書き込み系は従来どおり通常接続のままとし、挙動変更を広げない。

## 注意

- これは「常時ロックを避けやすくする」変更であり、SQLite の読み取り瞬間まで完全に無ロックにするものではない。
- `*.wb` を別DBへ分離しない限り、書き込みタイミングの排他は残る。
- WhiteBrowser互換を優先し、`*.wb` のスキーマや中身は変更しない。
