# AI向け 大機能詳細理解書（workthree 全体インデックス）

最終更新日: 2026-03-11

## 1. 目的

この資料は、`workthree` で AI が迷わず着手できるように、読む順番と主戦場を固定するための入口である。

## 1.5 このブランチで先に読む資料

この `workthree` ブランチでは、大機能理解より先に以下を読むこと。

- `AI向け_ブランチ方針_workthreeユーザー体感テンポ最優先_2026-03-11.md`
- `AI向け_ブランチ方針_future難読動画実験線_2026-03-11.md`
- `Thumbnail/現状把握_workthree_失敗動画検証と本線反映方針_2026-03-11.md`
- `Thumbnail/優先順位表_workthree_失敗9件の検証順_2026-03-11.md`

当面の目標は以下の 2 つで固定する。

1. ユーザーが感じるテンポ感を最優先で守り、改善する
2. UI の詰まりを解消し、一覧表示、キュー投入、サムネイル作成のテンポを上げる

難読動画検証は `future` の実験結果を参照しつつ、通常動画の高速感を壊さない範囲で本線へ返す。

## 2. 主に触る領域

- `MainWindow.xaml.cs` / `MainWindow.xaml`
  - 一覧更新、絞り込み、選択同期、UI テンポ改善
- `Watcher/`
  - 監視、再描画、補修判定
- `Thumbnail/`
  - キュー投入、サムネイル生成テンポ、関連ドキュメント
- `src/IndigoMovieManager.Thumbnail.Queue/`
  - Queue の実行状態、フォールバック、観測ログ

## 3. 先に読む順序

1. `AI向け_ブランチ方針_workthreeユーザー体感テンポ最優先`
2. `現状把握_workthree_失敗動画検証と本線反映方針`
3. `優先順位表_workthree_失敗9件の検証順`
4. 必要になった時だけ UI / Watcher / Queue / Engine 実装を読む

## 4. 改修時の最低チェックリスト

- ユーザーが最初に触る一覧、選択、最初の数件表示が軽いか
- 一覧更新で無駄な全件コピーや再代入を増やしていないか
- UI スレッドへ重い処理を戻していないか
- 難読動画対応が通常動画の既定経路を重くしていないか
- `future` 由来の変更を、一般条件へ圧縮して説明できるか

## 5. 更新時の記録ルール

- 対象資料の「最終更新日」を更新する
- 高速化本線向けか、難読動画実験線向けかを明記する
- 実装と文書の差分を同じコミット系列で揃える
