## 4. タスクの流し方

1. 調整役が対象タスクの指示書を担当エージェントへ渡す
2. 実装役は対象外レーンへ広げず、指示書の範囲だけを変更する
3. 差分が出たら Claude / Opus へレビュー専任で回す
4. 調整役が review finding を仕分けし、差し戻しまたは受け入れを決める

## 5. 受け入れゲート

- T1
  - MainDB read/write の入口候補が、呼び出し元別に整理されている
  - UI 直叩き、worker 直叩き、watch 用 writer が分かれている
- T2
  - `Factory + Interface + Args` 境界を壊していない
  - handoff 判定が App / worker 直書きへ戻っていない
- T3
  - `visible-only gate`
  - deferred batch
  - UI 抑制
  - 上の 3 条件の整合を壊していない
- R1
  - finding first
  - 重大度順
  - バグ、逆流、テンポ悪化、テスト不足を含む
