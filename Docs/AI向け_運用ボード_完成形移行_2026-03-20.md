## 4. 現在の運用モード

- 調整役は Codex に固定する
- 調整役は大きな実装を抱え込まず、次だけに専念する
  - 帯の切り分け
  - 実装役への配布
  - レビュー専任役への差分投入
  - findings の再配分
  - 受け入れ判断
- commit 作業は調整役が直接行わず、受け入れ済み帯だけを commit 専用サブエージェントへ渡す
- 実装役はレーン単位でのみ変更する
- レビュー専任役は finding first を守る

## 6. タスクの流し方

1. 調整役が対象タスクの指示書を担当エージェントへ渡す
2. 実装役は対象外レーンへ広げず、指示書の範囲だけを変更する
3. 差分が出たら Claude / Opus へレビュー専任で回す
4. 調整役が review finding を仕分けし、差し戻しまたは受け入れを決める
5. 受け入れ済み帯だけを commit 専用サブエージェントへ渡し、staging / commit / 身元確認を実施する
