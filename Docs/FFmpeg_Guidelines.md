# FFmpeg 利用ガイドライン（OSSとしての利用について）

動画管理ソフトにおけるサムネイル作成で `ffmpeg.exe` を利用する際の、ライセンス・特許に関する制限および運用方針のまとめです。

## 1. 基本的な利用方針

本ソフトウェアでは、ライセンスおよび特許のリスクを最小限に抑えるため、以下の構成を採用します。

*   **外部プロセス呼び出し**: FFmpegのコードをソフト内に組み込む（静的・動的リンク）のではなく、OSのコマンドとして `ffmpeg.exe` を外部から呼び出します (`Process.Start` 等による実行)。
*   **LGPL版の採用**: x264等のGPLライブラリを含まない、**LGPL版** のFFmpegを使用します。
    *   **最大のメリット**: あなたのソフトのソースコードを **非公開（プロプライエタリ）にしたまま配布できる** 点です。GPL版を使用した場合に生じる「ソフト全体のソースコードを公開しなければならない義務（いわゆるGPL汚染）」を防ぐことができます。
    *   **機能面の制限（エンコード不可・デコード可）**: x264 / x265（H.264, H.265エンコーダー）などのGPL成分が含まれないため、これらの形式での**動画エンコード（出力・変換）はできません**。ただし、サムネイル作成に必要な**動画のデコード（読み込み）と画像（JPEG/PNG等）出力は可能**であるため、本件の用途においては全く問題ありません。

### 推奨されるFFmpegのバイナリ
BtbNリポジトリ等から取得できる、以下のファイルが最適です。
*   **ファイル名**: `ffmpeg-n8.0-latest-win64-lgpl.zip` （または安定板の最新）
*   **理由**:
    *   `n8.0` などのリリース版（Stable）は動作が安定しています。
    *   `shared` ではない（静的リンク版）ため、DLL地獄に陥ることなく `ffmpeg.exe` 単体で動作し、配布や管理が容易です。

## 2. LGPL利用に伴う義務（必須対応）

LGPL版の `ffmpeg.exe` を同梱して配布する場合、以下の対応が必須となります。

1.  **ライセンス文の同梱**: `LICENSE.txt`（LGPL v2.1 または v3 のライセンス全文）を同梱する。
2.  **著作権の明示**: アプリの「ヘルプ」「バージョン情報」「README」等に、以下のように明記する。
    *   「本ソフトウェアはFFmpegを使用しています（ライセンス：LGPL）」
    *   FFmpeg公式サイトまたはソースコード配布元へのリンクURL。
3.  **リバースエンジニアリングの許可例外**:
    *   自社ソフトのEULA（利用規約）に「いかなるリバースエンジニアリングも禁止する」と記載がある場合、LGPLと矛盾します。
    *   「FFmpeg部分については、デバッグや改変のためのリバースエンジニアリングを許可する」といった例外条項を設ける必要があります。

## 3. 特許に関する注意点

*   H.264やH.265などの動画をデコード（読み込み）する場合、厳密にはMPEG-LAなどの特許が絡みます。
*   一般的なデスクトップアプリでのサムネイル作成などで大きなリスクになることは稀ですが、有料の動画配信サービスや、大規模な配布・ハードウェアへの組み込みの場合は特許料の確認が必要です。
*   出力（サムネイルとしての .jpg や .png 保存）自体には特許料は発生しません。

## 4. 実装上の推奨構成とコマンド

*   **配置**: インストールフォルダ内の `bin/ffmpeg.exe` などに同梱し、実行ファイルの場所を基準とした「相対パス」で呼び出します。
*   **高速なサムネイル生成コマンド**:
    時間を指定する `-ss` オプションを、入力ファイル `-i` の**前**に配置することで、ファイルの全スキャンを避け一瞬でシークさせることができます。

```shell
ffmpeg -ss 00:05:00 -i "input.mp4" -vframes 1 -f image2 "out.jpg"
```
