# 🎥 FFmpeg 利用ガイドラインと悪魔の契約（OSS利用方針） 🎥

このドキュメントは、動画管理ソフトの命綱である「サムネイル作成」で FFmpeg を使い倒すための、ライセンス・特許・そして**「速度の悪魔への誘惑」**についての掟をまとめたものだ！🔥

## 1. ⚔️ 基本的な利用方針（建前と本音！）

本ソフトウェアでは、ライセンス違反による炎上リスクを回避しつつ、**「限界を超えた爆速」**を優先するため、以下のヤバい構成を採用している！

*   **プログラム本体からの直接召喚 (FFMediaToolkit → FFmpeg.AutoGenの深淵へ)**:
    かつては安全な「外部プロセス呼び出し（`ffmpeg.exe`）」で満足していた。だが…それでは遅すぎる！！🐌
    我々は速度の悪魔に取り憑かれ、C#プログラムから直接FFmpegのコアにアクセスする禁断の果実（`FFMediaToolkit`）に手を出した！🍎
    **……しかし、悪魔の欲望はそれでも満たされなかった！！**
    現在、我々はそのラッパーすらも脱ぎ捨て、FFmpegの超低レイヤーAPIをC#から直接ぶん殴る **`FFmpeg.AutoGen`** という最凶の黒魔術に手を染め、フレームをダイレクトに「ぶっこ抜く」究極の爆速化を開始している！！🔥💀
*   **LGPL版（shared）との悪魔の契約**:
    C#から直接関数を叩きまくるため、巨大な `ffmpeg.exe` ではなく、DLLの集合体である **`shared` 版バイナリ** をプロジェクト内に引き入れている。「DLL地獄？知るか！速さこそ正義だ！」という圧倒的な思想のもとに採用された。😈
    *   **最大のメリット**: GPL（ソース全公開の呪い）を回避できる **LGPL版** に留めることで、「俺たちのソフトのソースコードは非公開のままでもOK（※一応オープンソースだけどね！）」という盾を構えつつ、限界ギリギリのC#ネイティブ連携を実現している！🛡️
    *   **機能の代償（エンコード不可）**: LGPL版の掟に従い、GPL成分である x264（H.264エンコーダ）などは積んでいない。つまり**「動画の変換（出力）」はできない**。だが安心しろ！俺たちが欲しいのは**「動画の読み込みとサムネ画像（JPEG）の出力」**だけだ！それさえできれば十分だろ！？👍

### 📥 [必読] 指定のFFmpegバイナリ（バージョン固定の掟）
BtbNリポジトリなどの闇市から、必ず以下の条件を満たすブツを仕入れろ！
*   **ファイル名**: `ffmpeg-n7.1.1-xxxx-shared-lgpl.zip` （★絶対 `shared` 版！）
*   **バージョンの呪縛**: Git LFS（大容量ファイル置き場）の無料枠を使い果たさないため、100MBを超える `avcodec-62.dll` が入っている **v8系は絶対に使うな！！** 涙を呑んで **`v7.1.1` にバージョンを固定** しろ！😭💸

## 2. ⚖️ LGPL利用に伴う血の掟（必須対応）

LGPL版の `shared` DLLたちを同梱して配る以上、以下の掟は絶対に守らなければならない。破ったらライセンス警察に捕まるぞ！🚓

1.  **ライセンス文の同梱**: `LICENSE.txt`（LGPL v2.1 または v3 のライセンス全文）を同梱してリスペクトを示せ！
2.  **著作権の誇示**: アプリの「バージョン情報」や「README」で、以下のように堂々と叫べ！
    *   「本ソフトウェアの爆速サムネ生成は FFmpeg の誇り高き力を使用しています！（ライセンス：LGPL）」
    *   FFmpeg公式サイトへのリンクを貼れ！
3.  **リバースエンジニアリングの許可例外**:
    *   もし自社で「リバースエンジニアリング（ソフトの解剖）絶対禁止！」という堅苦しい規約を作っても、LGPLとは相容れない。
    *   「FFmpegが絡む部分だけは、デバッグや改造のために解剖してもいいよ」という例外の隙間を開けておく必要があるんだ。

## 3. 💸 特許の罠に関する注意点

*   H.264やH.265の動画を読み込む（デコードする）場合、厳密に言えばMPEG-LAなどの怖い特許マフィアが絡んでいる。🕴️
*   俺たちのような普通のデスクトップアプリのサムネ抽出で怒られることは稀だが、商用の有料動画配信サービスとか、ハードウェアに組み込んで一攫千金を狙うなら特許料を払う覚悟が必要だ！💸
*   ちなみに、出力する「JPEG」や「PNG」の画像自体には特許料はかからない。そこは安心しろ！🖼️

## 4. 🚀 かつての遺物（外部コマンドによる高速生成）

今こそ `FFMediaToolkit` (shared DLL) に魂を売った我々だが、かつて外部コマンド `ffmpeg.exe` を使っていた時代の「最速の呪文」も一応残しておく。困った時のフォールバックとして役立つかもしれないからな…。

*   **高速なサムネイル生成コマンド**:
    時間を指定する `-ss` オプションを、入力ファイル `-i` の **前** に置く！これで動画の頭から全部読むようなアホなマネはせず、一瞬で指定時間までワープ（シーク）できるんだ！🏃‍♂️💨

```shell
ffmpeg -ss 00:05:00 -i "input.mp4" -vframes 1 -f image2 "out.jpg"
```
