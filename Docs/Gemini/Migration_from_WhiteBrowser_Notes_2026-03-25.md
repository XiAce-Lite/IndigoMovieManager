# ⚠ WhiteBrowserからの移行・並行利用時の注意 ⚠ (2026-03-25)

やっほー！ジェミニだよ！✨
WhiteBrowser（以下WB）からこのIMMに乗り換えようとしてる君、あるいは両方使い分けたい君へ！
いくつか「命より大事な注意点」があるから、作業前に必ず目を通してね！🔥

---

## 1. 🛡 絶対にDBファイルをコピーしてからコピー側を使うこと！
今のIMMはまだ絶賛開発中なんだ。WBのDB（*.wb）を直接開くこともできるけど、万が一に備えて**「別の場所にコピーしたDB」**を使うようにしてね！

- **バックアップは正義！** 壊れてから泣いても遅いぜ！😭
- コピーしたDBを使えば、WB側の環境を汚さずにIMMを試せるから安心・安全だよ！

## 2. 🌍 Unicode（絵文字・特殊文字）の「二重登録」問題
ここが一番のハマりポイント！ WBとIMMでは文字の扱いが全然違うんだ。

- **WBの場合**: ユニコード文字が使えないから、絵文字や特殊文字は別の文字に化けて保存されているよ。
- **IMMの場合**: 正しいユニコード文字として認識・登録するよ。

### 😱 何が起きるの？
同じ動画ファイルでも、WBで開いた時とIMMで開いた時で「パス（名前）が違う」と判定されちゃうんだ。
その結果……
1.  **二重登録になる**: IMMでスキャンすると、WBで化けていた項目とは別に、「正しい名前の項目」が新しく作られちゃうよ。
2.  **WBで壊れる**: IMMで登録された「正しい名前」の項目がWBのDBに書き込まれると、WB側ではその動画が見れなくなったり、エラーになったりする可能性があるよ。

**だからこそ、「DBをコピーして使う」のが鉄則なんだ！☝✨**

## 3. 🖼 コピー側を使うことでサムネイルの上書きを防止
サムネイルは基本的には共通の場所を見にいくけど、IMMがWBの作ったサムネを「書き換えちゃう」可能性があるんだ。

- **手塩にかけた「ベストシーン」が……！**: WBのベストシーン機能でこだわって作ったオリジナルサムネイルも、IMMが「おっ、サムネ古いな？新しく生成しなきゃ！」って気を利かせて上書きしちゃう可能性があるんだ。
- **WBに戻った時に困るかも？**: WBが想定してない形式で上書きされたり、お気に入りのシーンが消えちゃうと悲しいよね。
- **解決策**: これもDBファイルをコピーして別名（あるいは別フォルダ）で保存してコピー側を開けば、IMMは新しいサムネイルフォルダをその下に自動で作るから、WB側のサムネフォルダ（と大事なベストシーン）は無傷で済むよ！どや！😎✨
- **爆速だから安心！**: 「サムネまた作り直すの面倒だな……」って思わなくて大丈夫！IMMのサムネイル作成術式は文字通り**爆速**。1万個程度のサムネなら、アニメ見ている間にあっという間に作り終わっちゃうぜ！🚀☕✨
