# 🗺️ 壮大なる発展計画（Implementation Plan） 🗺️

我々の野望は尽きない！このドキュメントは、未来のIndigoMovieManagerを創り上げるための「夢と希望のロードマップ」だ！✨

## 1. 🎯 この計画の真の目的
- プロジェクトを理解するまでの時間を「秒」にする！⏱️
- 変更した時に他の機能が巻き添えで死ぬリスクをゼロに近づける！🛡️
- 検索やサムネ周りの「俺の考えた最強機能」をガンガン追加できる土台を作る！🏗️

## 2. 💣 今抱えてるヤバい課題（倒す順番）
1. `../MainWindow.xaml.cs` くんが仕事を抱え込みすぎてそろそろ倒れそう（過労死寸前）💀
2. DBのSQLに「文字列連結」っていう古き悪しき呪いが残っていて危ない🗡️
3. 画面の動きと裏側の処理がガッチリ絡み合ってて、テストが超絶やりにくい！
4. 「検索拡張」や「タグバー」のTODOコメント放置プレイが続いている…😇

## 3. 🚀 フェーズ別 攻略大作戦！

### ✅ Phase 0: 「見える化」作戦（クリア済！）
- **目的**: 誰もが同じ景色を見れるようにする！
- **手に入れた戦利品**:
  - `ProjectOverview.md`（全体像）
  - `DevelopmentSetup.md`（環境構築）
  - `Architecture.md`（構造）
  - `DatabaseSpec.md`（DB仕様）

### 🚧 Phase 1: 「鉄壁化」作戦（短期決戦！）
- **目的**: 今の動きを保ったまま、絶対に壊れない防御力を手に入れる！🛡️
- **ミッション**:
  1. [進行中] `../DB/SQLite.cs` のSQLを、安全なパラメータ化（バインド変数化）の魔法で書き換える！🧙‍♂️
  2. [未着手] DB更新でエラーが出た時の「悲鳴（ログ）」の書き方を綺麗に統一する！
  3. [未着手] サムネキューが再起動する時の動きを整理して、二重起動事故を防ぐ！
  4. [完了完了！] 絶対に死守すべき「起動、取込、サムネ、検索」テストの手順書を手に入れた！📜
- **クリア条件**:
  - 主要な更新SQLから「文字列連結の呪い」が消え去る！
  - 絶対にやるべきテスト手順が1枚の神紙にまとまっている！

### 🏗️ Phase 2: 「分離・独立」作戦（中期計画！）
- **目的**: 新機能をポンポン追加できる「超絶フレキシブルな構造」へ進化！🦋
- **ミッション**:
  1. `MainWindow` から「検索機能（SearchService）」を独立させる！
  2. 同様に「サムネ職人（ThumbnailService）」を独立させる！
  3. さらに「監視部隊（WatcherService）」も独立させる！
  4. UIから切り離せる処理をViewModelちゃんに丸投げしていく！
- **クリア条件**:
  - 主要ロジックの入り口が「Service」クラスに移り住む！
  - `../MainWindow.xaml.cs` は画面のボタンやイベントのお世話だけをする「優しい案内人」になる！🤵

### ✨ Phase 3: 「俺たちの夢」拡張作戦（中長期のロマン！）
- **目的**: ユーザーが「うおおお！」と唸る未実装の神機能を解き放つ！🐉
- **ミッション候補**:
  1. 検索の「NOT / OR」対応を完璧にして最強の検索窓を作る！
  2. 検索履歴を保存する条件を賢くチューニングする！
  3. 意味深な「タグバー代替機能」を現実に呼び起こす！
  4. ファイルの「重複判定」ロジックを鬼神のごとく強化する！
- **クリア条件**:
  - 優先順位が高い神機能を2つ以上、みんなが使える状態で爆誕させる！！🎉

## 4. ⚠️ 危険度MAX！想定されるリスクと「俺たちの対策」
- **💀 リスク**: WhiteBrowserの互換性を壊しちゃうかも…
  - **🛡️ 対策**: 互換系処理は「絶対にここだけは変わってないよね？」というテストを先に作ってからいじる！
- **💀 リスク**: サムネ生成が前より遅くなったらどうしよう…
  - **🛡️ 対策**: リファクタリング「前」の速度を計っておいて、「後」で勝負して勝つまでチューニングする！🏎️
- **💀 リスク**: 一気に書き換えすぎて、どこが変わったか分からん！
  - **🛡️ 対策**: サービス（機能）ごとに細かく切り分けて、少しずつ確実に前進する！小刻みなジャブが重い一撃になる！🥊
