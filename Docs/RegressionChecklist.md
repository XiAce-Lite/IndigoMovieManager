# 回帰チェック手順（Phase 1-4）

## 1. 目的
- 主要導線（起動、取込、サムネ、検索）の回帰を短時間で確認する
- 変更後の破壊的影響を、手動で最低限検知できるようにする

## 2. 前提
- `Debug|Any CPU` でビルド成功済み
- 小規模なテスト用動画フォルダを用意済み（5〜20ファイル程度）
- テスト用DB（新規または既存）を1つ用意済み

## 3. クリティカルチェック

### 3.1 起動・終了
1. アプリを起動する
2. メイン画面が表示される
3. 必要に応じてDBを開く
4. アプリを終了する

期待結果:
- 起動時にクラッシュしない
- 終了時にエラーが出ない
- 次回起動でウィンドウ位置/サイズ、前回DB設定が復元される

### 3.2 DBオープン・一覧表示
1. DBを開く
2. タブ（Small/Big/Grid/List/5x2）を順に切り替える
3. 先頭アイテム選択時に詳細ペインを確認する

期待結果:
- 一覧が表示される
- タブ切替で固まらない
- 詳細ペインのタイトル・パス・タグが表示される

### 3.3 監視フォルダ取込
1. 監視フォルダを1件設定（watch=ON）
2. アプリ起動中に対象拡張子の動画をフォルダへ追加
3. 一覧と件数表示を確認する

期待結果:
- 新規動画がDBへ登録される
- 一覧へ反映される
- 監視イベントで異常終了しない

### 3.4 サムネイル生成
1. 一覧で対象動画を選択
2. 「等間隔サムネイル作成」を実行
3. 進捗通知と表示更新を確認

期待結果:
- サムネイル画像が生成される
- エラー画像のまま残らない（対象ファイルが存在する場合）
- 生成中にUIが極端に不安定にならない

### 3.5 検索・履歴
1. 検索ボックスでキーワード検索
2. ヒット件数表示を確認
3. Enterで履歴登録
4. ドロップダウンから履歴選択
5. Deleteキーで履歴削除

期待結果:
- 検索結果が絞り込まれる
- 履歴が登録/選択/削除できる
- IME入力中に不正な検索暴発が起きない

### 3.6 タグ操作
1. タグ追加/編集/削除を実行
2. タグクリック検索（単体/Ctrl併用）を確認

期待結果:
- タグ変更が画面とDBに反映される
- タグクリックで意図した検索になる

### 3.7 再生・ブックマーク
1. 通常再生を実行
2. ブックマーク追加を実行
3. ブックマークから再生
4. ブックマーク削除

期待結果:
- プレイヤー起動に失敗しない
- score/view_count/last_date が更新される
- ブックマーク一覧が更新される

### 3.8 ファイル操作
1. リネーム実行
2. 必要に応じてコピー/移動/削除を実行
3. 一覧とサムネイル追従を確認

期待結果:
- DBの `movie_name` / `movie_path` が更新される
- 画面表示が破綻しない
- 監視有効時でも致命的エラーにならない
