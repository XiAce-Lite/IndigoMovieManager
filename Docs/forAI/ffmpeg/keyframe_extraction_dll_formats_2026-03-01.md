# 🎯 動画形式別！DLL（FFMediaToolkit等）を使ったキーフレーム（Iフレーム）抽出テクニックまとめ

コマンドラインツール（ffmpeg.exe）ではなく、C#から直でFFmpegのDLL（libavcodec等）を叩いてサムネをぶっこ抜く方法をまとめるよ！🔥
`FFMediaToolkit` や `FFmpeg.AutoGen` を使う場合、コンテナ形式によって読み書きのクセが全然違うから要注意だよ！✨

## 🚀 共通の基本戦略：DLLならではの高速シーク
DLL版の強みは、プロセス起動のオーバーヘッドがなく、メモリ上で直接フレームを展開できること！

### ① 高速タイムスタンプ抽出（`ReadVideoFrame` メソッドなど）
指定した時間に近いキーフレームまでシークしてから、順番にデコードして目的のフレームを取り出すよ！
```csharp
// 擬似コード的なイメージ
var file = MediaFile.Open(@"input.mp4");
// 55秒の地点へシーク（内部的には直前のキーフレームに飛ぶ）
file.Video.TrySeek(TimeSpan.FromSeconds(55));
// 目的の時間までデコードを空回しして、ぴったりのフレームを取得！
var frame = file.Video.GetNextFrame(); 
```

### ② Iフレーム限定の連続抽出（サムネ量産）
シークバー用のサムネを作る時は、フレーム単位でデコードしながら `IsKeyFrame` プロパティ（またはピクチャタイプがIフレームかどうか）を判定するよ！
```csharp
while(file.Video.TryGetNextFrame(out var frame)) {
    if(frame.IsKeyFrame) {
        // キーフレーム（Iフレーム）だけ保存！爆速！
        SaveImage(frame);
    }
}
```

## 🎬 コンテナ・コーデックごとの罠とDLL特有の解決策

DLL経由だと、ffmpeg.exeが裏で勝手にやってくれていた「よしなに」が効かなくなるから、以下の形式ごとの違いに気をつけてね！

### 📦 MP4 / MOV (H.264, HEVC)
* **特徴:** メタデータがしっかりしてるからシークが超正確！
* **注意点:** 可変フレームレート（VFR）の場合、タイムスタンプ（PTS/DTS）の計算をミスるとフレームが飛んだり重複したりするよ😇
* **最強手順:** 抽出時はフレームインデックス番号ではなく、必ず `TimeSpan` などの時間（PTS）ベースでシーク位置を指定してね！👍

### 📦 MKV (Matroska)
* **特徴:** いろんなストリームが混ざり合うカオスな箱👑
* **注意点:** 映像ストリーム（`VideoStreams`）が複数あると、目的と違うストリームをデコードしちゃう罠がある！
* **最強手順:** DLLで開いたら、必ず `file.VideoStreams.FirstOrDefault(v => v.CodecName == "h264" || v.CodecName == "hevc")` みたいな感じでメインの映像ストリームをガッチリロックオンすること！

### 📦 AVI / FLV (H.263, レガシーH.264など)
* **特徴:** 古のフォーマット。たまにインデックスが壊れてる👴
* **注意点:** DLLから `TrySeek` を呼んだ瞬間に例外（エラー）を吐くか、全然違う時間に飛んでいくことがあるよ！💥
* **最強手順:** これらの形式を開く時は、シークを一切使わず「先頭から順番に全部のフレームを律儀に読み込む（デコードしていく）」のが一番安全な力技！

### 📦 TS (MPEG-TS)
* **特徴:** 放送用！途切れに強いけど、ファイルの最初と最後に「目次（Durationなど）」がないことが多い😱
* **注意点:** DLLで開いた瞬間、動画全体の長さ（`Duration`）が `0` や `Unknown` になっていて、パーセント指定のシークができない！
* **最強手順:** タイムスタンプ指定でシーク自体はできるけど、長さに依存した処理を書かないこと！エラー耐性が低いDLL実装だとパケットロスで死ぬから、できれば事前のMP4変換（別プロセス）を挟むのが無難だね😇

### 🎥 ProRes RAW / 特殊な中間コーデック
* **特徴:** 全フレームが独立（All-Intra）してるプロ用形式🐘
* **注意点:** `frame.IsKeyFrame` を判定するコードを入れても、全部キーフレーム扱いになって一生間引きが終わらないよ！
* **最強手順:** そもそもキーフレーム判定をあきらめて「〇秒ごとに1枚デコードする」という固定間隔（モジュロ演算やタイムスタンプの足し算）のロジックに切り替えよう！✨

DLLだとプログラマーの腕が試される部分が大きいけど、この違いを押さえとけばどんな変態ファイルもC#から自由に操れるよ！最高じゃん！🚀
