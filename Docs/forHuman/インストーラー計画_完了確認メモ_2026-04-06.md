### 2.4 実機 install で見つかった追加修正

- 2026-04-06 に、`.NET 8 Desktop Runtime` が既に入っている端末で setup 実行時に `0x80070005` を確認した
- 原因は、Burn が `DotNetDesktopRuntime` を per-machine package として `C:\ProgramData\Package Cache\` へ keep キャッシュしようとしたため
- bundle 側を `Cache="remove"` に変え、既導入時に不要な常駐キャッシュを作らないよう修正した
