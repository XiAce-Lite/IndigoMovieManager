# 本家PR文面案 エンジン外だしと公開ミラー運用 2026-04-07

## 1. 想定タイトル

`engine 関連 source の外だしと public release 導線の整理`

## 2. 先に短く伝える版

Public repo から `Contracts / Engine / FailureDb / RescueWorker` の tracked source を外し、
build / release は private package と worker artifact consume を正本にする構成へ寄せました。

これにより、public 側の Git tree 由来 `Source code (zip/tar.gz)` に engine source が入らない形へ進めつつ、
preview / installer / mirror 運用を成立させています。

本家側で追加の複雑な準備は基本不要で、
差分確認のうえ取り込んでもらえれば運用できる想定です。

## 3. PR 本文案

### 概要

- Public repo から以下の tracked source を外しました
  - `src/IndigoMovieManager.Thumbnail.Contracts`
  - `src/IndigoMovieManager.Thumbnail.Engine`
  - `src/IndigoMovieManager.Thumbnail.FailureDb`
  - `src/IndigoMovieManager.Thumbnail.RescueWorker`
- Public 側 build / release は、private package と worker artifact を consume する前提へ整理しました
- 配布は `asset-only mirror` を正本に寄せ、public tag の自動生成 source zip を正式配布物として使わない運用へ寄せています

### 背景

問題だったのは GitHub Release asset の zip ではなく、
public tag 時点の Git tree から自動生成される `Source code (zip/tar.gz)` に engine source が入ってしまうことでした。

そのため、公開物から engine source を見せない要件に対しては、
public Git tree から tracked source を外すことが直接の対策になります。

### 今回の変更で揃えたこと

- public repo から engine 関連 tracked source を除去
- private package / worker artifact consume 前提の build 導線
- 公開ミラー fallback を含む GitHub Actions release 導線
- installer bundle の prerequisite 導線
- installer の live proof

### proof

- public zipball 確認:
  - `master` の commit `1909191` で、`Contracts / Engine / FailureDb / RescueWorker` が zipball に含まれないことを確認
- preview / workflow proof:
  - `24032239061` `private_engine_run_id=23997659256` で成功
  - `24032342401` 入力なし `workflow_dispatch` で成功
  - `24032500601` `private_engine_release_tag=v1.0.3.5` 指定で成功
- installer proof:
  - 2026-04-07 に `clean install` 成功
  - 2026-04-07 に `1.0.3.4 -> 1.0.3.5` upgrade 成功

### 補足

- installer 生成 script は、現在 checkout 中の csproj 版数ではなく、
  実際に包む package 内 `IndigoMovieManager.exe` の `FileVersion` を正本にするよう修正しています
- WiX build 前に `installer/wix/bin` と `installer/wix/obj` を掃除して、
  stale manifest により bundle 版数がずれる事を避けています

### 本家側にお願いしたいこと

- 基本は差分レビューのみです
- 取り込み後、public 側で engine source を持たない運用へ寄せる方針に問題がないか見てもらえれば十分です
- 追加で private repo 側の秘匿情報設定を本家にお願いする想定は、現時点では強くありません
