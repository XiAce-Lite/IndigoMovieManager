## 2. 今の判断

- Public repo の通常 build は、`artifacts/private-engine-packages/Release/private-engine-packages.props` を入口に Private packages consume 前提へ寄せる。
- worker は `artifacts/rescue-worker/publish/Release-win-x64` の marker / sync metadata を正本にし、Public source 直読みに戻さない。

## 3. このタスクで潰すもの

- `GitHub Release` の asset zip と、GitHub 自動生成の `Source code (zip/tar.gz)` は別物である。
- 問題になっていたのは asset zip ではなく、**tag 時点の Git ツリーを固める自動生成 source zip** である。
- その直接原因だった以下の tracked source を、Public repo から外す。

  - `src/IndigoMovieManager.Thumbnail.Contracts`
  - `src/IndigoMovieManager.Thumbnail.Engine`
  - `src/IndigoMovieManager.Thumbnail.FailureDb`
  - `src/IndigoMovieManager.Thumbnail.RescueWorker`

- 併せて、Public repo の build / release script も package / artifact metadata consume 前提へ切り替える。

## 4. 今の運用方針

- Public repo 単体 source build には戻さず、Private package / worker artifact consume を正本として扱う。
