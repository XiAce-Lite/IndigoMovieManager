# 設計メモ: 下部タブ分割方針 (2026-03-15)

## 実装状況メモ (2026-03-15)

- Phase 1 として `BottomTabs/ThumbnailProgress` は作成済み。
- Phase 2 として `BottomTabs/Debug/MainWindow.BottomTab.Debug.cs` を作成し、Debug タブの表示制御、ログプレビュー、操作イベントをここへ集約した。
- `BottomTabs/Bookmark/MainWindow.BottomTab.Bookmark.cs` も作成し、Bookmark の読込、再読込窓口、追加、削除をここへ集約した。
- `BottomTabs/Extension/MainWindow.BottomTab.Extension.cs` も作成し、詳細ペインの表示・非表示・選択追従をここへ寄せた。
- `BottomTabs/Extension/ExtensionTabView.xaml` / `.xaml.cs` も追加し、`Extension` も host 経由で `ExtDetail` を扱う形へ揃えた。
- `BottomTabs/SavedSearch/MainWindow.BottomTab.SavedSearch.cs` も作成し、保存済み検索条件タブの placeholder 表示の入口をここへ固定した。
- `BottomTabs/Common/MainWindow.BottomTabs.Common.cs` も作成し、下部タブ共通の選択制御とタブ切替時のシェル処理をここへ寄せた。
- `BottomTabs/Common/BottomTabActivationGate.cs` を追加し、`Bookmark / Extension / SavedSearch / Debug / ThumbnailProgress` の「非アクティブ時は更新せず、dirty だけ立てて、再表示時に 1 回だけ反映する」方針を揃え始めた。
- `Bookmark` / `SavedSearch` / `Debug` は、タブ内容の XAML も `UserControl` へ分離した。`MainWindow.xaml` 側は `LayoutAnchorable` の殻だけを持つ。
- `BottomTabs/ThumbnailProgress/ThumbnailProgressTabView.xaml` / `.xaml.cs` も追加し、`ThumbnailProgress` の XAML も `UserControl` へ分離した。
- 現時点では、下部タブは `MainWindow.xaml` 側を host の殻だけにし、各タブの中身は `BottomTabs/<TabName>` 側へ寄せている。
- 「全体の ON/OFF は上位、細かい表示判定はタブ側」という境界は、`ThumbnailProgress` で先に固定した。

## 1. 目的

- 下部タブまわりの保守性を上げる。
- `MainWindow.xaml` / `MainWindow.xaml.cs` に散っている下部タブ固有ロジックを、タブ単位で追える形にする。
- 大改修ではなく、`workthree` の体感テンポを壊さず段階的に移せる構成にする。

## 2. 現状の問題

- 下部タブの XAML 定義が `MainWindow.xaml` に集中している。
- 下部タブ固有のロジックも `MainWindow.xaml.cs` に混在している。
- 特に `サムネイル進捗` は、表示、設定、タイマー、差分更新、初期化が1か所へ寄りすぎている。
- `DebugTab` のように独自の表示制御を持つタブもあり、将来タブ追加時の見通しが悪い。

## 3. 大方針

- `MainWindow` は「シェル」に寄せる。
- 下部タブ固有の View / 状態 / 更新制御は、各タブのフォルダへ寄せる。
- ただし最初から重い MVVM へ寄せず、既存の partial class と UserControl を活かす。
- 共通化は最小限にして、「各タブが自分の責務だけ持つ」形を優先する。

## 5. 各フォルダの責務

### 5.1 `BottomTabs/Common`

- 下部タブ共通の ID と最低限の活性判定だけを置く。
- ここにビジネスロジックは置かない。

### 5.2 `BottomTabs/<TabName>`

- そのタブの View
- そのタブの表示更新条件
- そのタブ固有のイベント処理
- そのタブ固有の読み込み処理

他タブの状態は原則知らない。

## 6. MainWindow に残すもの

- Dock 全体のレイアウト
- `DockingManager` / `LayoutRoot` / ペイン自体の構成
- タブ生成と初期接続
- アプリ全体のライフサイクル
- タブ共通の最上位 ON/OFF

つまり `MainWindow` は「親」、各下部タブフォルダは「子」にする。

## 7. タブごとの現実的な切り分け

### 7.1 詳細タブ

- 既存 `ExtDetail` は活かしつつ、外側に `ExtensionTabView` を置く。
- `MainWindow` は host だけを触り、詳細表示本体は `BottomTabs/Extension` 配下で追える形にする。

### 7.2 ブックマークタブ

- `GetBookmarkTable()` と `BookmarkList.Items.Refresh()` 周辺をまとめる。
- 読み込みと UI 操作をこのタブの partial に寄せる。

### 7.3 保存済み検索条件タブ

- まだ薄いので、最初は「置き場だけ作る」でよい。
- 将来機能追加時に `MainWindow.xaml.cs` へ戻さない受け皿にする。

### 7.4 サムネイル進捗タブ

- 最優先の分割対象。
- ここへ以下をまとめる。
  - タイマー tick
  - 差分更新要求
  - `UpdateThumbnailProgressSnapshotUi()`
  - 「表示中だけ反映する」細かい判定
  - 設定 UI の同期

### 7.5 Debug タブ

- 既に独自性が強い。
- `ApplyDebugTabVisibility()` とログプレビュー更新を独立させると読みやすくなる。

## 8. 命名規則

- partial class: `MainWindow.BottomTab.<TabName>.cs`
- View: `<TabName>TabView.xaml`
- 表示制御: `<TabName>TabPresenter.cs` または `<TabName>TabController.cs`
- 活性判定: `<TabName>TabVisibilityGate.cs`

`Manager` や `Service` は広すぎるので避ける。

## 9. 段階的な移行順

### Phase 1

- `BottomTabs/ThumbnailProgress` を新設
- サムネ進捗の UI 更新関連をここへ分離
- `MainWindow.xaml` 側はまだ直置きでもよい

### Phase 2

- `BottomTabs/Debug`
- `BottomTabs/Bookmark`
- `BottomTabs/Extension`

### Phase 3

- `BottomTabs/SavedSearch`
- 共通ヘルパの最小整理

## 10. 実装上の注意

- XAML の `x:Name` と `ContentId` は急に変えない。
- まずはコード移動で責務境界を作る。
- View の完全分離は、その後でよい。
- 一気に全タブを触らず、1タブずつ完結させる。

## 11. 第一候補

最初の着手対象は `サムネイル進捗` タブ。

理由:

- 下部タブの中で一番ロジック量が多い
- 「表示中だけ UI 反映」の調整対象がここにある
- ここを切れると、下部タブ分割の型をそのまま他タブへ横展開できる

## 12. 結論

- フォルダは `BottomTabs` 配下に切る。
- その下にタブごとのフォルダを置く。
- `MainWindow` はシェルに寄せ、細かい調整は各タブフォルダの partial / presenter に寄せる。
- 最初の分割対象は `BottomTabs/ThumbnailProgress` とする。
