# 調査結果（UIボトルネック解消）workthree 参照版

最終更新日: 2026-03-11

## 1. この文書の位置づけ

- 本書は `future` で進めた UI 詰まり解消の要点を、`workthree` でも参照できるようにした控えである。
- `workthree` の主目標は難読動画検証であり、UI 最適化そのものは主戦場ではない。

## 2. future 側で確定した要点

- メイン一覧の重さの本命は、仮想化不足ではなく全件再構築・全件再バインドだった。
- 効いた初手は `FilteredMovieRecs` への一本化と、`ItemsSource` 再代入 / `Items.Refresh()` の撤去だった。
- 次点として、`FilterAndSort` 内の同期 `SELECT *` を UI スレッド外へ逃がした。

## 3. workthree での扱い

- この知見は本線へ返す候補として保持する。
- `workthree` 側で同じ変更を追うのは、難読動画検証を邪魔しない範囲に限る。
- UI 改善コミットを `workthree` へ入れる時は、難読動画再現性や FailureDb 比較を壊さないかを先に確認する。

## 4. 当面の判断

- `workthree` では、UI 改善は参考資料として扱う
- 難読動画検証に直接効かない UI 改修は後回しにする
- 本線へ返す時に必要になったら、この文書を入口に `future` 側コミットを個別判定する
