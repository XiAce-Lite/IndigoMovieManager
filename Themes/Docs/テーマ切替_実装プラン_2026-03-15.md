# 🎨 オリジナルカラー復元＆テーマ切り替え機能の追加計画

## 📝 Goal Description
ユーザーからの「設定で切り替えて元に戻せるように、別ファイルにオリジナルカラーを保存しておいてほしい」という要望に応えるよ！
現在のアプリは [App.xaml](file:///c:/Users/na6ce/source/repos/IndigoMovieManager_fork_workthree/App.xaml) に `BaseTheme="Inherit"` を直書きしているため、常にOSの色設定に連動してダーク/ライトが切り替わっちゃう状態。
これを改善して、**「OS連動（推奨）」「オリジナル（過去のDarkBlue等の固定色を再現）」** の2つのテーマを設定画面から選べるように実装するね！🔥

## 🔍 User Review Required
> [!IMPORTANT]
> 「オリジナル」テーマを選んだ場合は、以前の「`DarkBlue`（上部タブ）」「`Black`（サムネイル背景）」「`White`（上部検索結果の文字色）」などのベタ塗りの色が復活するようにするね。
> もし「ここは別の色にしたい！」などの微調整の希望があれば、実装後にでも教えてね！🥰

## 🛠️ Proposed Changes

### 1. 設定値の追加 ([Properties/Settings.settings](file:///c:/Users/na6ce/source/repos/IndigoMovieManager_fork_workthree/Properties/Settings.settings))
テーマを記憶するための設定値を追加するよ。
#### [MODIFY] [Settings.settings](file:///c:/Users/na6ce/source/repos/IndigoMovieManager_fork_workthree/Properties/Settings.settings)
- `ThemeMode` (string型: `"OsSync"` または `"Original"`) などの設定を追加（既定値は `"OsSync"`）

### 2. オリジナルテーマ用ResourceDictionaryの作成
過去のハードコードされていた色（DarkBlue, Black, White等）をリソースとして一つのXAMLファイルに分離・保存しておくよ！
#### [NEW] [OriginalColors.xaml](file:///c:/Users/na6ce/source/repos/IndigoMovieManager_fork_workthree/Themes/OriginalColors.xaml)
- `<SolidColorBrush x:Key="UpperTabBackground" Color="DarkBlue" />`
- `<SolidColorBrush x:Key="ThumbImageBackground" Color="Black" />`
- `<SolidColorBrush x:Key="HeaderForeground" Color="White" />`
などのBrushを定義。

### 3. テーマの動的切り替えロジック実装
アプリ起動時や、設定変更時に動的にスタイルを適用するロジックを組むよ。
#### [MODIFY] [App.xaml](file:///c:/Users/na6ce/source/repos/IndigoMovieManager_fork_workthree/App.xaml)
- `MaterialDesign:BundledTheme` や動的なResourceDictionaryの読み込み制御の基盤を準備。
#### [MODIFY] [App.xaml.cs](file:///c:/Users/na6ce/source/repos/IndigoMovieManager_fork_workthree/App.xaml.cs)
- 起動時に `Settings.Default.ThemeMode` を読み取り、テーマ（MaterialDesignのLight/Dark または OriginalColors.xaml の読み込み）を適用するメソッド `ApplyTheme(string mode)` を追加。
- `OS連動` 時は `BaseTheme.Inherit` に任せず、OSのダーク判定から `Light / Dark` を明示選択する。
- `AvalonDockTheme` も `OS連動` 時は `Vs2013LightTheme / Vs2013DarkTheme` をコード側で切り替える。
- `OS連動` 中は `SystemEvents.UserPreferenceChanged` を購読し、Windows 側のテーマ変更を受けたら即時 `ApplyTheme("OsSync")` を再実行する。

### 4. MainWindowのXAML修正
ハードコード削除で対応した各箇所に対して、追加する「テーマリソース」の `DynamicResource` 参照を埋め込むよ。（オリジナルテーマ時には色がつき、OS連動時には透明や標準色になるようにする）
#### [MODIFY] [MainWindow.xaml](file:///C:/Users/na6ce/source/repos/IndigoMovieManager/Views/Main/MainWindow.xaml)
- 上部タブの背景: `Small / Big / Grid / 5x2` は `Background="{DynamicResource UpperTabBackground}"` を使う
- `List` タブは色指定を持たず、既定のテーマ表示へ任せる
- `ListDataGrid.RowStyle` は暗黙の `DataGridRow` スタイルを潰さないよう `BasedOn="{StaticResource {x:Type DataGridRow}}"` を付ける
- 画像の背景: `Background="{DynamicResource ThumbImageBackground}"`
- ヘッダ文字色: `Foreground="{DynamicResource HeaderForeground}"`
- メインヘッダーは背景だけ専用キーを使い、文字色は `App.ApplyTheme()` の `MainHeaderForegroundBrush` / `MainHeaderInputForegroundBrush` で切り替える
- `Original` のメインヘッダー背景は、初期コミット時の `MaterialDesignThemes 4.9.0 / Indigo / PrimaryDark` 相当の `#FF303F9F` を使う
- `SearchBox` / `ComboSort` は `MaterialDesignOutlinedComboBox` を使い、検索履歴項目のスタイルも `MaterialDesignComboBoxItemStyle` 継承でMDIX既定を維持する
- `ComboSort` は `MainHeaderCompactComboBoxStyle` で高さを 32 に抑え、メインヘッダー内だけ密度を上げる
- 検索欄は `MainHeaderSearchBoxStyle` で高さを 36、`Reload` は `MainHeaderButtonStyle` で高さを 32 に揃え、`ColorZone` の余白と `DockingManager.Margin` も合わせて詰める
- 標準タイトルバーは `DwmSetWindowAttribute` で制御し、`OsSync` ダーク時は immersive dark、`Original` は `#FF303F9F + White` の caption/text 色を明示する
- 左ドロワー: 視認性を落とさないため `NavigationDrawerBackground` を使って白背景を維持する
- 左ドロワー文字色は `LeftDrawerForegroundBrush` で切り替え、OS連動では本文色、`Original` では `#FF303F9F` を使う
- 上側タブの文字パネルは `UpperTabPanelForegroundBrush` で切り替える
- `OS連動ダーク` のパネル文字は `DarkGray` 固定、`OS連動ライト` は `MaterialDesignBody`、`Original` は `SystemColors.ControlTextBrush` に寄せる
- `Grid` の動画名だけはテーマ分岐から外し、全テーマで少し暗めの固定グレーを使う
- パネル背景のテーマ分岐は一旦外し、`Grid` のタイトル帯は従来どおり半透明黒、`Small / Big / Big10` の文字面は背景なしへ戻す
※これらは `OriginalColors.xaml` が読まれている時だけ有効に（あるいは OS連動用リソースディクショナリも用意してFallback等で対応）なるように調整。

## ✅ Verification Plan

### Manual Verification
1. **アプリ起動**: アプリを起動後、「OS設定と連動（OSがダークならダーク）」になっていることを確認する。
2. **設定画面の操作**: 設定画面を開き、テーマを「オリジナル」に変更する。
3. **即時反映の確認**:
    - 上部タブの背景が `DarkBlue` になるか。
    - サムネイル画像の背景が `Black` になるか。
    - 各種文字色が `White` 等の指定色になるか。
4. **再起動後の維持**: 一度アプリを落として再起動した際、設定した「オリジナル」テーマが維持されているか確認する。
5. **「OS連動」への戻し**: 再び設定画面から「OS連動」に戻し、ダーク（またはライト）テーマのデフォルト色に戻るか確認する。
