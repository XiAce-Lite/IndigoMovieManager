# DCO（エンジン分離実装規則 2026-03-05）

## 1. 目的
- サムネイル Engine / Queue 分離後の実装で、層の逆流や責務混在を防ぐ。
- 仕様追加時に「どこへ実装すべきか」を明確化し、後退を防止する。

## 2. 適用範囲
- `src/IndigoMovieManager.Thumbnail.Engine/**`
- `src/IndigoMovieManager.Thumbnail.Queue/**`
- `Thumbnail/Adapters/**`
- `Thumbnail/MainWindow.*.cs`（呼び出し側としての遵守）

## 3. DCO本体（必須規則）

### DCO-01 層境界（UI型の持ち込み禁止）
- Engine / Queue 側に `WriteableBitmap` / `ImageSource` / `Dispatcher` / `MessageBox` を持ち込まない。
- UI表現が必要な場合は、Engine/Queue では中立なDTO（値）だけ返す。

### DCO-02 参照方向（一方向依存）
- 依存方向は `App -> Engine/Queue` の一方向を維持する。
- Engine/Queue から App プロジェクト参照を追加しない。
- Engine/Queue で必要な機能は Interface を定義し、App 側 Adapter で実装する。

### DCO-03 UI更新フロー（逆流禁止）
- UI更新イベントは App 層で完結させる。
- Engine へ UI コールバックを逆流させない（`Action<UI型>` の注入禁止）。
- Engine/Queue は進捗や結果を「UI非依存の通知データ」で返す。

### DCO-04 QueueDB契約（拡張子と責務）
- QueueDB の実体ファイル拡張子は `.queue.imm` を正とする。
- 拡張子解決は PathResolver に集約し、呼び出し側で文字列直書きしない。
- Engine は QueueDB の物理パス規約を知らない（Queue 側に閉じる）。

### DCO-05 例外とログ
- Engine/Queue は UI 表示を行わず、例外送出または結果オブジェクトで通知する。
- ログは層内で完結する形式を使い、UIメッセージ文言を直接持たない。

### DCO-06 テスト境界
- Engine/Queue の単体テストは UI なしで実行できる状態を維持する。
- 境界破り（WPF型参照、App参照追加）が入った場合はマージしない。

## 4. 実装時チェックリスト（PR前）
- [ ] Engine/Queue に WPF 参照（`System.Windows.*`）を追加していない
- [ ] Engine/Queue から App 参照が増えていない
- [ ] UI更新は App 層のみで完結している
- [ ] QueueDB 拡張子 `.queue.imm` を直書きせず Resolver 経由になっている
- [ ] `Debug|x64` / `Release|x64` 以外の構成を増やしていない
- [ ] 影響範囲のテスト（Engine/Queue/App）を最低1本ずつ更新または確認した

## 5. 例外運用（どうしても破る必要がある場合）
- 例外は「一時措置」として期限付きで記録する。
- 例外追加時は以下を同時にコミットする。
  - 破る規則ID（例: `DCO-01`）
  - 理由
  - 期限
  - 戻しタスクID（Implementation Plan / tasklist にリンク）

## 6. 変更管理
- 本DCOを変更するPRは、変更理由を必ず `Implementation Plan` に残す。
- DCO改定前に、既存コードとの不整合を棚卸ししてから更新する。
