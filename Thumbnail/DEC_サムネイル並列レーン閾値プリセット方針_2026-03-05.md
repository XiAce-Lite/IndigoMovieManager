# DEC: サムネイル並列レーン閾値プリセット方針（2026-03-05）

## 1. ステータス
- 承認済み（Accepted）

## 2. 背景
- サムネイル生成で大動画が先に実行枠を占有すると、小動画の完了表示が遅れて体感が悪化する。
- 並列数だけを増やしても、動画サイズ分布次第で期待した改善が出ない。
- そのため、サイズ閾値でレーンを分け、プリセットで即時切替できる運用を標準化する。

## 3. 決定事項
- レーン閾値の設定項目を持つ。
  - `ThumbnailPriorityLaneMaxMb`（優先レーン上限MB）
  - `ThumbnailSlowLaneMinGb`（低速レーン開始GB）
- 閾値のクランプ範囲は以下とする。
  - 優先レーン上限: `50〜4096 MB`
  - 低速レーン開始: `1〜1024 GB`
- プリセットは以下3種で固定する。
  - 軽量動画重視: 優先 `128MB` / 低速 `1GB` / 並列 `論理コア数の1/2`
  - バランス: 優先 `512MB` / 低速 `3GB` / 並列 `論理コア数の1/3`
  - 巨大動画重視: 優先 `1024MB` / 低速 `10GB` / 並列 `論理コア数の1/4`
- プリセット適用時は、閾値だけでなく並列数も同時に即時反映する。

## 4. 意図
- 小動画の完了速度を上げ、進捗タブの体感レスポンスを改善する。
- 巨大動画の滞留を抑えつつ、全体スループットを維持する。
- ユーザーがワークロードに応じて、ワンクリックで挙動を切り替えられるようにする。

## 6. 注意点
- `Settings` の既存ユーザー値がある場合は、既定値よりユーザー値が優先される。
- 初回起動時にどのプリセット値を初期採用するかは、`Settings` 既定値と初期化順序の整合を確認して運用する。

## 7. 見直し条件
- 小動画完了のP95が悪化した場合
- 低速レーン滞留が常態化した場合
- 実運用でプリセット切替頻度が高く、追加プリセット要望が明確になった場合
