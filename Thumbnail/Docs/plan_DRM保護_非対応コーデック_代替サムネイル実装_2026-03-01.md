# DRM保護/非対応コーデック 代替サムネイル実装 計画兼タスクリスト（2026-03-01）

更新日: 2026-03-20

変更概要:
- WMV / ASF の DRM precheck を 1 GUID から 2 GUID 検出へ拡張
- `Content Encryption Object` と `Extended Content Encryption Object` の両方を対象化
- 判定詳細にヒットした GUID 種別を残すよう追記

## 1. 背景
- 一部動画で全サムネイルエンジンが失敗し、キューがFailedのまま残る。
- 代表例として、DRM保護疑い（例: PRDY系）や非対応コーデックでデコード不能なケースがある。
- 2026-03-18 追記: 0B動画は `CODEC NG` ではなく専用の `NO DATA` 表示へ分ける。
- 失敗時に専用画像をサムネイルとして出すことで、UI上の判別性と運用性を上げる。

## 2. 目的
- 全エンジン失敗時に、失敗理由を分類して専用プレースホルダー画像を生成する。
- プレースホルダー生成に成功した場合はサムネイル作成成功として扱う。
- 0B動画は `NO DATA` として識別できるようにする。

## 3. 実装方針
- 変更対象: `Thumbnail/ThumbnailCreationService.cs`
- 通常フロー（autogen -> ffmediatoolkit -> ffmpeg1pass -> opencv）は変更しない。
- 全エンジン失敗時のみ、エラー文言を集約して以下に分類する。
  - NO DATA（0B動画）
  - DRM保護疑い
  - 非対応コーデック/デコード不能
  - 判定不能（現状どおり失敗）
- 0B / DRM保護疑い / 非対応と判定できた場合、専用プレースホルダー画像を生成する。
- 生成画像は既存のサムネイル保存先（`SaveThumbFileName`）へ保存する。

## 4. 判定キーワード（初期版）
- DRM保護疑い:
  - `prdy`
  - `playready`
  - `drm`
  - `encrypted`
  - `protected`
  - `no decoder found for: none`
  - WMV / ASF 先頭64KB内の DRM GUID
    - `2211B3FB-BD23-11D2-B4B7-00A0C955FC6E` (`Content Encryption Object`)
    - `298AE614-2622-4C17-B935-DAE07EE9289C` (`Extended Content Encryption Object`)
- NO DATA:
  - 実ファイルサイズが `0`
  - `zero-byte`
  - `0 byte`
- 非対応コーデック/デコード不能:
  - `decoder not found`
  - `video stream not found`
  - `unknown codec`
  - `unsupported`
  - `invalid data found`

## 5. 受け入れ条件
- 0B / DRM保護疑い / 非対応ケースで、専用画像が作成される。
- プレースホルダー作成時は `CreateThumbAsync` が成功扱いで返る。
- 判定不能ケースは既存どおり失敗扱いとエラーマーカー出力を維持する。
- ビルドが通る。
