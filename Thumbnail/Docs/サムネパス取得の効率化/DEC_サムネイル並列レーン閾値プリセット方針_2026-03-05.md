# DEC: サムネイル並列レーン閾値プリセット方針（2026-03-05）

## 1. ステータス
- 承認済み（Accepted）
- 2026-03-28 更新: `優先レーン上限MB` は実装から外れ、現在は `低速レーン開始GB` と `並列数` の組み合わせで運用する。

## 2. 背景
- サムネイル生成で大動画が先に実行枠を占有すると、小動画の完了表示が遅れて体感が悪化する。
- 並列数だけを増やしても、動画サイズ分布次第で期待した改善が出ない。
- そのため、サイズ閾値でレーンを分け、プリセットで即時切替できる運用を標準化する。

## 3. 決定事項
- レーン閾値の設定項目は `ThumbnailSlowLaneMinGb`（低速レーン開始GB）のみを持つ。
- 閾値のクランプ範囲は `1〜200 GB` とする。
- プリセットは以下5種で運用する。
  - 低速: 低速 `50GB` / 並列 `2本固定`
  - 普通: 低速 `100GB` / 並列 `論理コア数の1/3`
  - 高速: 低速 `100GB` / 並列 `論理コア数の1/2`
  - SSD: 低速 `100GB` / 並列 `論理コア数`
  - MAX: 低速 `100GB` / 並列 `論理コア数の2倍`
- プリセット適用時は、低速レーン閾値と並列数を同時に即時反映する。

## 4. 意図
- 小動画の完了速度を上げ、進捗タブの体感レスポンスを改善する。
- 巨大動画の滞留を抑えつつ、全体スループットを維持する。
- ユーザーがワークロードに応じて、ワンクリックで挙動を切り替えられるようにする。

## 6. 注意点
- `Settings` の既存ユーザー値がある場合は、既定値よりユーザー値が優先される。
- 初回起動時にどのプリセット値を初期採用するかは、`Settings` 既定値と初期化順序の整合を確認して運用する。
- レーン分類は `Normal` / `Slow` の2系統で、優先レーン上限の別設定は持たない。

## 7. 見直し条件
- 小動画完了のP95が悪化した場合
- 低速レーン滞留が常態化した場合
- 実運用でプリセット切替頻度が高く、追加プリセット要望が明確になった場合
