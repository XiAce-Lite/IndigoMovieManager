# 優先順位表 workthree 失敗9件の検証順 2026-03-11

最終更新日: 2026-03-11

## 1. 目的

この文書は、`future` で進める難読動画検証結果を、`workthree` 本線へ取り込む時の優先順位を定めるための表である。

ファイル名に `失敗9件` とあるが、本書では個別動画名を扱わない。
本線では、一般化できた条件だけを優先度付きで受け取る。

## 2. 取り込み優先順位

| 優先 | 条件カテゴリ | 本線で先に検討する理由 | 取り込み条件 |
|---|---|---|---|
| P1 | 早期失敗確定できる条件 | 無駄な再試行を減らし、通常系のテンポを守りやすい | 失敗判定が明確で、通常動画に副作用が少ない |
| P2 | repair / preflight で軽く救える条件 | UI や Queue を大きく変えずに成功率を上げやすい | 実行コストが軽く、既定経路を重くしない |
| P3 | autogen の過渡失敗整理 | 既定エンジンの安定化は全体へ効く | retry / fallback 条件が比較ログで説明できる |
| P4 | パス条件や特殊文字条件 | 対象を限定しやすく、通常系へ波及しにくい | 条件判定が安価で限定的である |
| P5 | 重い難読動画専用の分岐 | 効果はあるが、通常系へ副作用が出やすい | レーン分離や優先度隔離まで含めて説明できる |

## 3. 本線へ返す時の必須情報

- 成功条件
- 導入位置
- 回帰確認ポイント
- 通常動画のテンポへ与える影響
- 追加ログや観測項目

## 4. 本線で後回しにするもの

- 個別動画名ベースの分岐
- UI テンポを悪化させる救済条件
- 比較ログなしの retry 増加
- 実験用ハーネス依存のままの実装
