# 絵文字パス4段階対策（方針・現状）

## 1. 目的
- 絵文字を含むファイル名・ディレクトリ名でもサムネイル生成を安定化する。
- 大容量動画のコピーを最後の手段に限定する。
- 失敗時の挙動をログで追跡できるようにする。

## 2. 段階定義
1. 生パス
2. 短縮名（`GetShortPathName`）
3. 別名（ジャンクション / ハードリンク）
4. 一時コピー（最終手段）

補足:
- 実装内部ではログ追跡のため、`JunctionAlias` と `HardLinkAlias` を別段階で扱う。

## 3. 採用フロー
1. OpenCV で 1,2,3 を試行
2. OpenCV が失敗した場合のみ ffmpeg で 1,2,3 を試行
3. それでも不可の場合のみ 4（コピー）へ進む
4. 3GB超コピーは後回しにしてユーザー確認後に実行

## 4. 実装済みポイント（2026-02-24）
- Raw/Short で失敗した場合のみ、段階3（別名）を作成する。
- OpenCV/ffmpeg ともに同じ順序で段階試行する。
- コピー段階は 3GB 超の場合 `Deferred` 返却し、通常キュー完了後に確認する。
- 保存時は直接 `ImWrite` が失敗・例外の場合でも、一時ASCII保存へ自動フォールバックする。
- ANSI判定は厳密変換（例外フォールバック）で実施する。
- `Encoding 932` 対策として `Encoding.RegisterProvider(CodePagesEncodingProvider.Instance)` を初期化時に登録する。

## 5. クリーンアップ方針
- 別名（ジャンクション/ハードリンク）と一時コピーは `finally` で必ず削除する。
- ジャンクションは再解析ポイントを辿らず削除する。
- クリーンアップ失敗はログ出力し、処理継続を優先する。

## 6. ログ方針
- `thumb opencv input try/selected` で試行段階と採用段階を出す。
- `thumb capture open failed` で存在判定つき失敗ログを出す。
- コピー遅延・コピー実行・クリーンアップ失敗を専用ログで出す。

## 7. 非対象
- `System.Windows.Data Error: 5 (MaxWidth が負値)` は本対策の対象外（UIレイアウト側）。
