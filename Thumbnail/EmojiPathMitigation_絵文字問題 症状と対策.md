# 😱 絵文字パス問題 症状と対策のまとめ 🛡️

動画ファイルやフォルダの名前に絵文字が入ってるとアプリが死んじゃう！？そんな恐怖の「絵文字パス問題」と、それを完全にねじ伏せるための対策まとめだよ！🔥

## 1. どんな症状が出るの？😭
- **動画が読めない！:** パスに絵文字（WindowsのANSI変換不可能なUnicode特有文字）が含まれてると、動画を正しく開けなくてサムネイル生成が爆発する💣
- **画像保存でコケる！:** サムネが作れても、保存先パスに絵文字があると書き込みエラーで泣く羽目に…📉
- **丸ごとコピー作戦の罠:** 「じゃあ安全な場所にファイルを丸ごとコピーしてから読もう！」ってすると、3GBとかの大容量動画が来た瞬間に処理が激遅になってPCのディスクも圧迫する激ヤバ事態に！😱

## 2. 犯人は誰だ！絵文字に弱いヤツら🕵️‍♂️
- **OpenCV (`OpenCvSharp4` など):**
  - 内部のC/C++処理やOSの連携部分で、マルチバイト文字（CodePage 932など）を前提にしてるせい。絵文字入りのパスを渡すと `VideoCapture` や `Cv2.ImWrite` が見事に文字化けして死ぬ！💀
- **古いシステム・一部の外部プロセス:**
  - 古き良き厳格なエンコーディング環境は大体アウト（※ちなみに `ffmpeg` たんは比較的絵文字に強いけど、絶対じゃないから過信は禁物！）。

### 2.1 調査結果：ffmpeg と .NET (C#) って実際どうなの？🤔
- **ffmpegへの引数渡し (セーフ！):**
  - `ProcessStartInfo.ArgumentList` を使って渡してるから、コマンドプロンプトの文字コード制限をすり抜けて安全にUnicodeのまま渡せる！ffmpeg自体もUTF-8/Unicodeパスを解釈できるからここはノープロブレム！😎
- **.NETネイティブのファイル操作 (超セーフ！):**
  - C# の `System.IO`（`File.Copy`, `File.Delete` など）は内部でちゃんとUnicodeとしてWindows APIを叩いてる。だから標準機能でファイルを動かしたり消したりする分には、絵文字があろうが全く問題なし！✨

---

## 3. 最強の対策：4段階防御＆保存フォールバック！🦸‍♂️
大容量ファイルの「丸ごとコピー」は極力避けて、OSの力を借りた**「4段階の入力パス対策」**と**「保存時の無敵フォールバック」**で戦うよ！⚔️

### 3.1 鉄壁の4段階入力パス対策（OpenCV / ffmpeg共通）
1. **生パス (Raw):** 勇気を出してまずはそのまま特攻！🔥
2. **短縮名 (ShortPath):** 失敗したら `GetShortPathName` APIの出番！絵文字を含まない「8.3形式の短いパス」を取り出して再アタック！💨
3. **別名 (Junction / HardLink):**
   - パスの「フォルダ部分」に絵文字があるなら、ASCII文字だけの一時フォルダを作ってそこから **ジャンクション (Directory Junction)** を張る裏技を発動！🔗（1,2失敗時のみ）
4. **一時コピー (Copy):**
   - 最終奥義！安全な場所にファイルを丸ごとコピーする！（ffmpegが完全に詰んだ時のみ）
   - **大容量対策:** もしファイルが3GBを超えてたら、アプリを固めないために一旦「後回し（Deferred）」状態にして、ユーザーの確認を経てから実行する親切設計！💖

### 3.2 保存時の無敵フォールバック処理 🛡️
- **ANSI変換判定と安全な保存:**
  `Encoding.GetEncoding(932)` の厳格なチェックで「パスに絵文字入ってる？」を判定。安全なら `Cv2.ImWrite` でそのままスッと保存！
- **一時ASCII保存と移動 (ここがミソ！):**
  絵文字が検知されたら、**一時的に適当なASCII名のパスに保存**しちゃう！その直後、Unicode完全対応の `.NET File.Move` を使って本来の絵文字パスの場所にシュッと移動！完璧すぎる！✨
  （※起動時に `Encoding.RegisterProvider(CodePagesEncodingProvider.Instance)` でCodePage 932の準備をしておくのがポイント！）

### 3.3 クリーンアップとロギング 🧹
- **立つ鳥跡を濁さず:** 一時的に作ったジャンクションやコピーファイルは、終わったら `finally` ブロックで絶対に消し去る！ジャンクションを消す時に本体まで消さないように気をつけてね！🥺
- **バッチリ記録:** どのパス対策で突破したのか、どこでコケたのかをログ（`thumb opencv input selected` とか）にしっかり残す！これで何かあってもすぐに原因を突き止められるぜ！！🕵️‍♂️👍
