# サムネイルエンジン比較ベンチ機能まとめ（2026-03-01）

対象コード:
- `Tests/IndigoMovieManager.Tests/ThumbnailEngineBenchTests.cs`
- `Thumbnail/Test/run_thumbnail_engine_bench.ps1`
- `Thumbnail/Test/run_thumbnail_engine_bench_folder.ps1`

## 1. 何ができるか

同じ動画ファイルを入力にして、サムネイル生成エンジンごとの処理速度と成功率を比較できる。

- 比較対象エンジン: `autogen`, `ffmediatoolkit`, `ffmpeg1pass`, `opencv`
- 各エンジンを同じ回数だけ実行
- 実行結果をCSV保存
- 実行後にエンジン別の平均時間/成功数を即時集計
- フォルダ一括版は実行前に残存ベンチプロセスを自動停止する
- フォルダ一括版は1ファイルごとのタイムアウトで自動スキップできる

## 2. 入力パラメータ

- `IMM_BENCH_INPUT`: 比較対象の動画フルパス（必須）
- `IMM_BENCH_ENGINES`: 比較エンジン指定（カンマ/空白/セミコロン区切り可）
- `IMM_BENCH_ITER`: 反復回数（既定3）
- `IMM_BENCH_WARMUP`: ウォームアップ回数（既定1）
- `IMM_BENCH_TAB_INDEX`: サムネイルレイアウトを決めるTabIndex（`4` で10パネル）

## 3. 出力

### 3.1 CSV

保存先:
- `%LOCALAPPDATA%\IndigoMovieManager\logs\thumbnail-engine-bench-<timestamp>.csv`

列:
- `datetime`
- `input_file_name`
- `engine`
- `iteration`
- `tab_index`
- `panel_count`
- `elapsed_ms`
- `success`
- `duration_sec`
- `output_bytes`
- `output_path`
- `error_message`

### 3.2 実行時サマリ

`run_thumbnail_engine_bench.ps1` 実行時に以下をエンジン別で表示する。
- `Runs`（試行数）
- `Success`（成功数）
- `Failed`（失敗数）
- `AvgMs`（平均処理時間）
- `MinMs`（最短）
- `MaxMs`（最長）

## 4. 実行方法

### 4.1 推奨（スクリプト）

```powershell
pwsh -File .\Thumbnail\Test\run_thumbnail_engine_bench.ps1 `
  -InputMovie "C:\path\to\movie.mp4" `
  -Engines "autogen,ffmediatoolkit,ffmpeg1pass,opencv" `
  -Iteration 3 `
  -Warmup 1 `
  -TabIndex 4 `
  -Configuration Debug `
  -Platform "x64"
```

### 4.1.1 フォルダ一括（複数取得版 / 10パネル）

```powershell
pwsh -File .\Thumbnail\Test\run_thumbnail_engine_bench_folder.ps1 `
  -InputFolder "$env:USERPROFILE\Desktop\読込困難ぶでー" `
  -Engines "autogen,ffmediatoolkit,ffmpeg1pass" `
  -Iteration 1 `
  -Warmup 1 `
  -TabIndex 4 `
  -GpuMode "cuda" `
  -FileTimeoutSec 300 `
  -Configuration Debug `
  -Platform "x64"
```

出力:
- combined: `logs\thumbnail-engine-bench-folder-tab4-combined_yyyyMMdd_HHmmss.csv`
- summary: `logs\thumbnail-engine-bench-folder-tab4-summary_yyyyMMdd_HHmmss.csv`

### 4.2 直接 `dotnet test` で実行

```powershell
# 先にMSBuildでビルド（COM参照対策）
"C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" `
  .\IndigoMovieManager.sln `
  /p:Configuration=Debug `
  /p:Platform=x64 `
  /m

$env:IMM_BENCH_INPUT = "C:\path\to\movie.mp4"
$env:IMM_BENCH_ENGINES = "autogen,ffmediatoolkit,ffmpeg1pass,opencv"
$env:IMM_BENCH_ITER = "3"
$env:IMM_BENCH_WARMUP = "1"
$env:IMM_BENCH_TAB_INDEX = "4"
dotnet test .\Tests\IndigoMovieManager.Tests\IndigoMovieManager.Tests.csproj `
  -c Debug `
  --no-build `
  --filter "FullyQualifiedName~ThumbnailEngineBenchTests.Bench_同一入力でエンジン別比較を実行する"
```

## 5. 運用メモ

- 入力動画が存在しない場合、ベンチテストは `Ignore` になる。
- 失敗の詳細はCSVの `error_message` 列で確認する。
- エンジン強制用の `IMM_THUMB_ENGINE` は、ベンチ実行中だけ上書きし、終了時に復元する。
- `-Engines` は `"autogen,ffmediatoolkit,ffmpeg1pass"` のほか、`"autogen ffmediatoolkit ffmpeg1pass"` や `"autogen;ffmediatoolkit;ffmpeg1pass"` でも指定できる。
- `run_thumbnail_engine_bench_folder.ps1` は開始時に残存ベンチプロセスを自動killする。
- `run_thumbnail_engine_bench_folder.ps1` は `-FileTimeoutSec` を超えたファイルを自動スキップし、`combined` に `failed`（`timeout skip`）として記録する。

## 7. 結果まとめ方法（だれでも同じ形式で作れる）

### 7.1 summary をそのまま使う

`summary` CSV を開けば、最低限の比較結果（runs/success/failed/avg/min/max）は揃っている。  
提出時は以下を固定で記載する。

1. 条件（フォルダ/エンジン/Iteration/Warmup/TabIndex/GPUモード）
2. `summary` のフルパス
3. `combined` のフルパス
4. エンジン別の比較表
5. 失敗件数と代表エラー

### 7.2 combined から「遅い上位3件」を抽出

```powershell
$combinedCsv = Join-Path (Get-Location) "logs\thumbnail-engine-bench-folder-tab4-combined_yyyyMMdd_HHmmss.csv"
Import-Csv $combinedCsv |
  Where-Object { $_.success -eq "success" } |
  Sort-Object {[double]$_.elapsed_ms} -Descending |
  Select-Object -First 3 input_file_name,engine,elapsed_ms,tab_index,panel_count
```

### 7.3 failed の要因を抽出

```powershell
$combinedCsv = Join-Path (Get-Location) "logs\thumbnail-engine-bench-folder-tab4-combined_yyyyMMdd_HHmmss.csv"
Import-Csv $combinedCsv |
  Where-Object { $_.success -ne "success" } |
  Select-Object input_file_name,engine,error_message
```
