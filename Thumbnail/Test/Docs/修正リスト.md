## 今回確定した仕様（反映必須）
- `UpdateStatus` は「リース所有者一致時のみ更新」。
- `ResetFailedToPending` は `AttemptCount = 0` にリセットする。
- タブ切替時は既存DBキューを保持する（破棄しない）。
- DB切替時も既存DBキューは保持し、Consumerは「現在開いているDB」のみ処理する。
- `GetPendingAndLease` は `preferredTabIndex` 指定時に対象タブを優先してリースする。
- `Upsert` は `Status=Processing` 行を上書きしない（処理中リースを破壊しない）。
- 旧 in-memory 重複管理（`queuedThumbnailKeys`）は廃止し、重複抑止はQueueDB一意制約を正とする。
- Producer側で同一キー連打をデバウンスし、Persister側で同一キーをバッチ内圧縮する（二段階抑制）。
- Phase 5で `enqueue/persisted/lease/failed` のメトリクスをログ出力する。
- 手動再試行は `ResetFailedThumbnailJobsForCurrentDb` を運用入口とする。
