# 黒多め背景専用救済方針

## 目的

- 通常救済 route には混ぜず、手動指定時だけ「黒多め背景」の動画を深掘りする
- near-black reject だけでは落ちるが、一部に明るいコマがある個体を拾いやすくする

## 現在の実装

- rescue request の `ExtraJson` に `rescue_mode=dark-heavy-background` を載せられる
- rescue worker はこの mode を読むと、通常成功時でも near-black retry を強制実行する
- 対象 engine は次の 3 つ
  - `autogen`
  - `ffmpeg1pass`
  - `ffmediatoolkit`
- 右クリックメニューから `黒多め背景救済` を明示実行できる
- 右クリックメニューから `黒多め背景救済Lite` も明示実行できる

## 専用ロジック

- 通常 near-black retry より候補点数を増やす
- 通常 long 尺の候補
  - `10% / 35% / 65% / 85%`
- 黒多め背景 mode の候補
  - `3% / 5% / 8% / 12% / 18% / 25% / 35% / 50% / 65% / 80% / 90%`
- 超短尺でも候補点数を増やす
  - `3% / 5% / 8% / 10% / 12% / 15% / 20% / 25% / 35% / 50% / 70% / 85%`

## Lite の役割

- `dark-heavy-background-lite` は「より良いコマを探す」より「まず 1 枚返す」を優先する
- 候補点数は `dark-heavy-background` と同じだけ増やす
- ただし near-black 候補でも reject し切らず、最初に取れた 1 枚を返しやすくする
- 真っ黒回避の深掘りは通常版、まず 1 枚返すのは Lite と使い分ける

## 境界

- 通常 route には自動適用しない
- `CODEC NG` 固定や `NoVideoStream` 判定を上書きしない
- まずは manual-only の指定 mode として使う

## 次の拡張候補

- 進捗タブへ `RescueMode` を表示する
- 下側 `サムネ失敗` タブや上側 `救済` タブからも mode 指定できる UI を足す
- 明るさ以外に、顔検出やエッジ量の重みを mode 専用で上げる
