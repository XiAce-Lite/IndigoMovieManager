# AI向け オーバーレイ表示利用ガイド 2026-03-30

最終更新日: 2026-03-30

変更概要:
- UI hang 通知とユーザー要請結果通知の overlay 入口を整理
- 色分け、行数、表示高さ、使い分けを AI 向けに明文化
- サムネ系ユーザー要請は modal より overlay を優先する現方針を明記

## 1. この資料の目的

- AI が overlay 通知を触る時に、どの入口を使うべきかを迷わないようにする
- 色や行数の判断を都度バラさず、既存ルールに沿って変更できるようにする
- `MessageBox` に戻してしまう事故や、UI hang 通知の既定挙動を壊す事故を防ぐ

## 3. 入口の使い分け

### 3.1 UI hang / 明示ステータス

使用入口:
- `_uiHangNotificationCoordinator.ShowExplicitStatus(...)`
- `_uiHangNotificationCoordinator.HideExplicitStatus()`
- `_uiHangNotificationCoordinator.ReevaluateVisibility()`

用途:
- DB切替中
- 終了中
- AI が一時的に明示したい overlay
- UI hang 系の通常通知

注意:
- `ShowExplicitStatus(...)` を直接使う時は、非表示タイミングもセットで設計する
- `allowBackground: false` なら、前面時だけ表示される
- `Critical` だけは背後でも出る

### 3.2 サムネ系ユーザー要請結果

使用入口:
- `ShowThumbnailUserActionPopup(...)`

用途:
- 右クリック系の `サムネ作成`
- `サムネ救済`
- `単処理サムネ救済`
- `インデックス再構築`
- 上側 `サムネ救済` タブの各種実行結果
- 下部 `サムネ失敗` タブの選択救済 / 一括救済

方針:
- ユーザー要請の結果通知は、まずこの helper を使う
- `IsLoaded` 前や coordinator 未初期化時だけ `MessageBox` へフォールバックする
- AI がサムネ系結果通知を追加する時は、原則この helper に寄せる

## 4. 色の意味

`UiHangNotificationLevel` は現在こう使う。

- `Success`
  - 受付成功
  - 開始成功
  - 反映成功
  - 文字色は緑
- `Caution`
  - 対象なし
  - busy
  - 既に実行中
  - 対象外
  - 既に正常サムネイルあり
  - 文字色は黄
- `Warning`
  - 不開始
  - 受理不可
  - 実行失敗寄りの案内
  - 文字色は赤
- `Critical`
  - UI hang の危険状態
  - 背後でも表示する特別扱い

サムネ系は `ResolveThumbnailUserActionOverlayLevel(...)` で文言から意味寄せしている。
新しい文言を追加する時は、この判定も合わせて更新すること。

## 5. 行数と高さ

現在の見た目ルール:

- 1行通知
  - 高さ `48`
- 複数行通知
  - 3行前提
  - 高さ `78`
  - 開始位置は少し下げている

補足:
- 実装上の line count は 1〜3 で扱う
- 成功通知は `タイトル / 成功文 / 件数や補足` の 3 行になりやすい
- 複数行時は縦中央寄せではなく、上から詰めて描画する

現状の主な定数:
- `OverlaySingleLineHeight`
- `OverlayMultiLineHeight`
- `OverlayMultiLineTextTopOffset`

定数の場所:
- `Views/Main/NativeOverlayHost.cs`

## 6. サムネ系 overlay 文言の組み立て

入口:
- `BuildThumbnailUserActionOverlayMessage(...)`

補助:
- `BuildSuccessThumbnailUserActionOverlayMessage(...)`
- `SplitThumbnailUserActionMessageLines(...)`
- `BuildThumbnailRescueUserActionPopupMessage(...)`
- `BuildThumbnailIndexRepairUserActionPopupMessage(...)`
- `BuildThumbnailQueueUserActionPopupMessage(...)`
- `BuildThumbnailBlackConfirmUserActionPopupMessage(...)`

考え方:
- まず popup message builder が意味単位の文言を返す
- 次に overlay builder が、色と行数に合わせて表示用メッセージへ整形する
- 成功通知は 3 行まで使って見やすさを優先する
- 注意・警告は情報量を残しつつ、過度に縦へ伸ばしすぎない

## 7. AI が守るべきルール

- ユーザー要請の結果通知は、勝手に `MessageBox.Show(...)` を直接増やさない
- overlay を触る時は、色・行数・高さをセットで確認する
- 成功文言だけを短くしたい時は、popup message builder から変える
- 見た目だけ詰めたい時は、`NativeOverlayHost.cs` 側だけで調整する
- `UI hang` 本流と `サムネ系ユーザー通知` を同じ意味で扱わない
- `Critical` の挙動は UI hang 用なので、通常のユーザー通知へ流用しない
