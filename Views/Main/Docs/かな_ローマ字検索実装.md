# かな・ローマ字検索実装 2026-04-07

## 目的

- 通常検索で `Kana` と `Roma` の両方を検索対象に含める。
- 新規登録、リネーム、旧DB補完の3経路で `kana` / `roma` をDBへ流し続ける。
- 旧バージョンDBで `kana` または `roma` が空でも、検索と後追い補完の両方で取りこぼさない。

## 今回の採用方針

### 1. `Kana` / `Roma` をDB正本として保存する

- `movie.kana` / `movie.roma` と `bookmark.kana` / `bookmark.roma` は既存列を使う。
- `kana` は元祖 WhiteBrowser に合わせて、DB正本をひらがな保存に統一する。
- `roma` は `JapaneseKanaProvider.GetRomaFromKana(...)` で `kana` から生成する。
- 保存導線は「新規登録」「リネーム」「backfill」で共通化する。

### 2. 検索時 fallback は残す

- 旧DBで `Kana` / `Roma` がまだ空でも、`Movie_Name` / `Movie_Path` から検索時に補完できるようにする。
- これにより backfill 完了前でも検索不能期間を作らない。

### 3. WhiteBrowser 互換寄りの検索しやすさを優先する

- 基本ローマ字は `kana -> romaji` の素直な変換を使う。
- ただし検索では長音違いを拾いやすくするため、簡略表記や IME 寄り alias も同時に持つ。

## 実装ポイント

### `Infrastructure/JapaneseRomajiConverter.cs`

- `Kana` から検索用ローマ字文字列を作る小さな変換器。
- 拗音、促音、長音を正規化し、検索向け alias を組み立てる。

### `Infrastructure/JapaneseKanaProvider.cs`

- `Windows.Globalization.JapanesePhoneticAnalyzer` を直参照し、漢字混じりの題名をひらがなへ寄せる。
- 既存の `GetKana(...)` を維持しつつ、`GetRoma(...)` と `GetRomaFromKana(...)` を追加した。
- `kana` 保存時はひらがな正規化し、検索時だけカタカナ比較キーを併用する。
- 日本語題名と英語副題が混在するファイル名では、音韻解析トークンから日本語部分だけを拾って `kana` / `roma` を作る。

### `DB/SQLite.cs`

- `ComputeReadingValues(...)` を追加し、DB保存前の `kana` / `roma` 生成を1か所へ寄せた。
- `InsertMovieTable(...)` と `InsertMovieTableBatch(...)` で `kana` / `roma` を条件付きINSERTする。
- `InsertBookmarkTable(...)` と `UpdateBookmarkRename(...)` でも `kana` / `roma` を保存する。
- 旧DB向けに列有無を毎回確認し、存在する列だけを安全に更新する。

### `Views/Main/MainWindow.KanaBackfill.cs`

- 既存のかな補完フローをそのまま使い、`roma` も同時に後追い補完するよう拡張した。
- `kana` だけ空、`roma` だけ空、両方空のいずれも補完対象にする。
- UI側の `MovieRecords.Kana` / `MovieRecords.Roma` も同時更新する。

### `Watcher/MainWindow.WatcherRenameBridge.cs`

- rename 時に `Movie_Name` / `Movie_Path` だけでなく `Kana` / `Roma` も再計算する。
- `MainDbMovieMutationFacade.UpdateRoma(...)` を追加し、DB単一列更新経路でも `roma` を扱えるようにした。

### `ViewModels/MainWindowViewModel.cs`

- 検索対象は `Movie_Name` / `Movie_Path` / `Tags` / `Comment*` / `Kana` / `Roma` を含む。
- `Kana` はひらがな正本を使いながら、カタカナ入力でもヒットする比較キーを追加する。
- `Roma` 列が空でも `Kana` から導出して検索に使う。

## 挙動

### 新規登録・リネーム

- 追加直後から `kana` / `roma` がDBに保存される。
- rename 後も `movie` / `bookmark` の表記と読みが同期する。

### 旧DB補完

- DBオープン後、空の `kana` / `roma` だけを少量ずつ補完する。
- `kana` が既にある行はそれを再利用して `roma` だけを埋める。

### 検索

- `Kana` が埋まっていればそのままヒットする。
- `Kana` がひらがな保存でも、カタカナ入力を比較側で吸収する。
- `Roma` が埋まっていればそれも検索対象になる。
- どちらかが空でも検索時 fallback により検索性を維持する。

## スコープ外

- SQL 側への検索 pushdown
- WhiteBrowser の全ローマ字流儀の完全再現
- `kana` / `roma` 専用 index の追加
