# 障害対応: オーバーレイ終了残留防止 2026-04-20

## 背景

- UI hang 通知の overlay は `Views/Main/NativeOverlayHost.cs` で別 STA thread と fallback 用 `Window` を使っている。
- 終了時は `BeginInvokeShutdown` を投げて 250ms だけ待つ形だったため、dispatcher 側の処理順や fallback window の close 競合次第で、補助 window が閉じ切る前に `MainWindow` だけ先に閉じる筋が残っていた。

## 今回の対応

- `NativeOverlayHost.Stop()` の thread join 待機を 1000ms へ広げた。
- join timeout 時は `Dispatcher.InvokeShutdown()` で強制 shutdown を追加し、shutdown 要求がキューで詰まっても overlay thread を抜けやすくした。
- `App.xaml.cs` の `OnStartup` で `ShutdownMode.OnMainWindowClose` を明示し、overlay fallback などの補助 window が最後の 1 枚として残ってもアプリ終了を妨げないようにした。

## 期待する効果

- MainWindow を閉じた後に、overlay 由来の隠し window が残ってプロセス終了を引っ張る確率を下げる。
- native/fallback どちらの経路でも、終了時は MainWindow close を最優先で完了できる。
