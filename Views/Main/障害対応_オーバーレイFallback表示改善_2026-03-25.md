# Overlay 表示不具合（UI詰まり通知）の対応記録

更新日: 2026-03-25

## 背景
`UiHangNotification` の通知表示で、native オーバーレイ経路（`UpdateLayeredWindow`）が失敗した後、
フォールバック Window は表示ログが出るものの実際に見えない状態が継続していました。
特に Windows の DPI/複数モニタ環境や座標算出時の境界条件で、座標が画面外に出るケースが疑われました。

## 対応内容

### 1) 表示経路をフォールバック優先に切替
- ファイル: `Views/Main/NativeOverlayHost.cs`
- `NativeOverlayHost` の `UpdateLayeredWindow` 失敗時、`_useFallbackWindow` を有効化して native 描画を停止し、
  WPF フォールバックへ即時移行するようにした。
- 初期化時の `EnsureLayeredWindowStyles` / `SetLayeredWindowAttributes` 失敗時も、
  起動時から fallback を有効化しておくようにした。
- これにより `error=87` 系ログの後、native 側で再試行し続ける状態を防いでいる。

### 2) fallback の位置補正を追加
- ファイル: `Views/Main/NativeOverlayHost.cs`
- `RenderOverlayOnOverlayThread` で座標算出後に `ResolveAndClampOverlayPosition` を呼び、
  表示先モニタの作業領域（`MONITORINFO.work`）にクランプするようにした。
- `RenderByFallbackWindow` では対象モニタの DPI を `GetDpiForMonitor` で取得し、DIP へ換算して `Left/Top/Width/Height` を設定するようにした。
- フォールバック Window は `WindowStartupLocation.Manual`、`Topmost` を再保証し、`Show` 失敗や位置ズレによる「見えない」状態を抑制。

## 変更した主なログ
- `overlay native initialize failed, fallback window mode enabled`
- `overlay native renderer disabled: switched to fallback window (error=...)`
- `overlay fallback show: ... x=... y=...`
- `overlay fallback render failed: ...`

## 確認手順
1. 起動後に `ui-tempo` 側で `level=Caution/Warning` が出ることを確認
2. `ui-overlay` ログで上記 fallback 系ログが出ることを確認
3. 以前は見えなくなっていた表示が実際に画面下中央付近へ出ることを確認
4. `ui-tempo` で `ui hang recovered` が出た際に `overlay hide request` で通知が消えることを確認
