# 要件定義（フォーム下タブ: サムネイル進捗）

## 1. 目的
- 目的は「作成スレッドの可視化」。
- サムネイル作成中の状況を、ユーザーが画面内で即時把握できるようにする。

## 2. スコープ
- 対象:
  - メインフォーム下部に新規タブ `サムネイル進捗` を追加する。
  - 左サイドに全体情報、右サイドにスレッド別パネルを表示する。
- 非対象:
  - `*.wb` スキーマ変更
  - サムネイル生成ロジックそのもののアルゴリズム変更

## 3. 画面要件（配置）

### 3.1 全体レイアウト
- 新規タブ名: `サムネイル進捗`
- 配置: フォーム下部のタブ領域
- 2カラム構成:
  - 左: 基本情報（固定幅）
  - 右: 現在スレッド数ぶんの作業パネル（可変、スクロール可）

### 3.2 左サイド（基本情報）
- 表示項目:
  - 作成済み数 / 総キュー
  - DB登録待ち件数 / DB総数
  - 現在スレッド数 / 最大スレッド数 / 論理コア数
  - CPUメーター（0-100%）
  - GPUメーター（0-100%、取得不可時は `N/A`）
  - HDDメーター（0-100%、取得不可時は `N/A`）
  - キュー投入ログ（動画名の最新数行）

### 3.3 右サイド（スレッド別パネル）
- パネル数: 「現在のスレッド数」と一致
- 各パネル表示:
  - 作業中の動画名（長い場合は中間省略、拡張子保持）
  - 作成直後の画像プレビュー（高さ統一の小サイズ）

## 4. 機能要件

### FR-01 タブ追加
- 下部タブに `サムネイル進捗` を追加し、起動中いつでも参照可能にする。

### FR-02 基本情報のリアルタイム更新
- 左サイド情報は定期更新する（初期値: 500ms 間隔）。
- UIスレッドをブロックしない。

### FR-03 キュー投入ログ
- 最新N件（初期値: 10件）をリングバッファで表示する。
- 1行は動画名のみを表示し、古い行から押し出す。

### FR-04 スレッド別パネル同期
- スレッド開始時にパネルを作成/更新し、終了時に `待機` 表示へ戻す。
- 同時実行数が変動したらパネル数も追従する。

### FR-05 動画名の省略表示
- 省略規則:
  - 末尾の拡張子は必ず保持
  - 例: `abcdefghijklmnopq...mp4`

### FR-06 直近画像プレビュー
- サムネイル保存直後の画像を各パネルへ表示する。
- 表示高さは固定（初期値: 72px）、横幅はアスペクト比維持で自動。

### FR-07 フェイルセーフ
- CPU/GPU/HDD取得失敗時でもタブ全体は表示継続する。
- 失敗項目のみ `N/A` にフォールバックする。

## 5. データ要件（値の出どころ）

| 項目 | 既存流用 | 追加実装 |
|---|---|---|
| 作成済み数 / 総キュー | `ThumbnailQueueProcessor` 内セッション値 | UI公開用 `ThumbnailProgressSnapshot` が必要 |
| DB登録待ち件数 / DB総数 | `PendingMovieRecs.Count` / `MovieRecs.Count` | 既存流用可 |
| 現在/最大スレッド数 | `currentParallelism` / 設定値 | `currentParallelism` のUI公開が必要 |
| 論理コア数 | `Environment.ProcessorCount` | 既存API流用 |
| CPUメーター | なし | プロセスCPUサンプラ追加 |
| GPUメーター | なし | GPU使用率サンプラ追加（取得不可時N/A） |
| HDDメーター | なし | ディスク使用率サンプラ追加（取得不可時N/A） |
| キュー投入ログ | なし | `TryEnqueueThumbnailJob` でリングバッファ追加 |
| スレッド別動画名/画像 | 一部ログのみ存在 | ジョブ開始/完了イベントとスナップショット保持が必要 |

## 6. 非機能要件
- UI更新間隔: 500ms（将来設定化可能）
- 追加UIのオーバーヘッド:
  - CPU常時 +2%以内を目標
  - メモリ増分は画像キャッシュ含め 100MB 以内
- 既存の `FilterAndSort`、一覧描画、再生操作を劣化させない。

## 7. 受け入れ条件
1. サムネイル処理中に `サムネイル進捗` タブで進捗が更新される。
2. 左サイドの全項目が更新され、取得不可項目は `N/A` で表示される。
3. 右サイドに「現在スレッド数」と同数のパネルが表示される。
4. 長い動画名が拡張子を残して省略表示される。
5. 各パネルで直近画像が高さ統一で表示される。
6. タブ表示が既存処理をブロックしない（操作遅延が体感増加しない）。

## 8. 実装フェーズ（提案）
- Phase A: データ収集層
  - 進捗スナップショット、キュー投入ログ、CPU/GPU/HDDサンプラを追加
- Phase B: ViewModel接続
  - 進捗タブ専用ViewModelを追加し、500msで更新
- Phase C: UI実装
  - 下部タブ追加、左サイド情報、右サイド動的パネルを実装
- Phase D: 回帰確認
  - 大量キュー時の表示追従、既存UI性能、例外時フォールバックを確認
