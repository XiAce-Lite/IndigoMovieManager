# スキン最小シェルと独立フレーム方針 2026-04-02

## 目的

- WhiteBrowser 互換の外部スキンを導入しても、既存アプリの安全な導線と戻り道を失わないようにする。
- 本家 WhiteBrowser 風の自由なヘッダー設計を許しつつ、IndigoMovieManager 側の責務を最小に保つ。
- 旧スキン互換と将来の自由度を両立する。

## 今回固定する結論

- メインヘッダーは「最小シェル」として残す。
- スキンは `MainWindow` 内の独立した表示フレームとして扱う。
- 実 OS の別トップレベル Window にはしない。
- host 側はブラウザの chrome に近い最小 UI 枠だけを持ち、skin ごとに on/off を切り替えられる形を目指す。
- 検索 UI はスキン側に置けるが、検索条件と検索結果の正本は本体側に置く。

## 背景

本家 WhiteBrowser 互換のカスタムスキンまで視野に入れると、メインヘッダーへ全操作を固定で載せる案は自由度を削る。
一方で host 側の UI を完全に消すと、設定、DB 切替、fallback、再読込、保守系導線が弱くなる。

このため、host 側は最小シェルだけを持ち、一覧画面の主役はスキンへ寄せる方針を採る。

## 採用する構造

### 1. host 側

- `MainWindow` 全体の殻
- DB 切替
- 設定
- `Grid` へ戻る
- スキン再読込
- 現在のスキン名表示
- `サムネ救済` / `重複動画` への導線

### 2. skin 側

- 一覧 UI
- 検索 UI
- ソート UI
- 絞り込み UI
- 独自ヘッダー
- 独自ツールバー

### 3. 本体側の正本

- 検索条件
- フィルタ結果
- ソート結果
- 選択状態
- DB 状態
- rescue / watcher / 下部タブとの整合

## 実装上の扱い

- 外部スキンは `ExternalSkinHostPresenter` に載せる。
- 既存 WPF 一覧は fallback と安全軸として残す。
- スキンは「別画面風」に扱うが、同じ `MainWindow` 内で切り替える。
- フォーカス、IME、ショートカット、選択同期を壊さないため、実 OS の別 Window 案は採らない。

## host chrome モード

### Host Chrome On

- host 側が検索、ソート、戻るなどを出す。
- 最小スキン、旧スキン、管理寄り運用向け。

### Host Chrome Minimal

- host 側はスキン名、戻る、再読込、設定など最小導線だけ出す。
- 外部 WB スキンの既定モード候補。

### Host Chrome Off

- host 側の一覧ヘッダーは出さず、スキンが全面管理する。
- 互換重視モード。
- ただし fallback 導線だけは別経路で必ず残す。

## 旧スキン互換の考え方

- 旧スキンが独自ヘッダーを持つ場合は、host chrome を `Minimal` または `Off` に寄せる。
- 旧スキンがヘッダーを持たない場合は、host chrome を `On` または `Minimal` にして不足 UI を補う。
- この構造なら、スキンごとに自由度を確保しながら、既存 host の戻り道も維持できる。

## 検索の扱い

- スキン側に検索 UI を置くことは採用する。
- ただし検索処理そのものは本体側で実行する。
- スキンは `wb.*` 相当の bridge で検索要求を送り、本体が `FilteredMovieRecs` などを更新し、スキンは再描画だけを担う。

## タブとの関係

- `スキン` は `TabItem` ではなく表示モードとして扱う。
- `Grid` は WPF 側の安全軸として残す。
- `Small / Big / List / 5x2` は将来的に内蔵スキン化を進める余地がある。
- `サムネ救済` と `重複動画` は一覧スキンとは別の業務タブとして維持する。

## 非採用案

### 1. スキンを専用 TabItem にする

- 一覧全体を差し替える設計と相性が悪い。
- 本家 WhiteBrowser 風の自由なヘッダーを作りにくい。

### 2. スキンを実 OS の別 Window にする

- フォーカス
- IME
- ショートカット
- 選択同期
- 下部タブ連携

上記の複雑性が増えすぎるため採らない。

## 次の実装判断

1. host chrome の最小 UI 案を決める
2. `Host Chrome On / Minimal / Off` の切替地点を定義する
3. 検索 bridge の契約を追加する
4. `5x2` を最初の内蔵スキン化候補として進める

## 一言で言うと

「host は最小シェル、skin は独立フレーム、正本ロジックは本体側」

この 3 点を守ることで、WhiteBrowser 互換の自由度と IndigoMovieManager の運用安全性を両立する。
