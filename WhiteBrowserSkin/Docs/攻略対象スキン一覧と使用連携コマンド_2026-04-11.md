# 攻略対象スキン一覧と使用連携コマンド 2026-04-11

## 目的

- 外部スキン互換を進める時に、どのスキンから攻略すると効率が良いかを固定する
- 各スキンが使っている `wb.*` 連携コマンドと callback を一覧化し、未実装や不足挙動を追いやすくする

## 対象スキン一覧

### リポジトリ同梱スキン

1. `skin/DefaultGridWB`
2. `skin/SimpleGridWB`

### テスト用 WhiteBrowser スキン

1. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/TutorialCallbackGrid`
2. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultBig`
3. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultGrid`
4. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultList`
5. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultSmall`

## 攻略優先順

1. `TutorialCallbackGrid`
   - callback が最も濃く、`onSkinEnter`、`onUpdate`、`onCreateThum`、`onSetFocus`、`onSetSelect`、`scrollSetting` をまとめて確認できる
2. `WhiteBrowserDefaultList`
   - `scroll-id` を持つため、一覧描画とスクロール領域の意味論確認に効く
3. `WhiteBrowserDefaultGrid`
   - 最小の `onCreateThum` 系グリッド描画として横展開しやすい
4. `WhiteBrowserDefaultSmall`
   - `score` 表示を含むため、旧 field alias の確認に向く
5. `WhiteBrowserDefaultBig`
   - `id` と `score` の見せ方が異なるため、描画差の確認に向く
6. `SimpleGridWB`
   - 既存互換ランタイムでの安定動作確認用ベースラインとして常時監視する

2026-04-11 進捗注記:

- `TutorialCallbackGrid` は実 fixture 統合テストで、初回表示と先頭再更新 clear を確認済み
- `WhiteBrowserDefaultList` は実 fixture 統合テストで、default `onUpdate` / `onCreateThum` / `scroll-id` の組み合わせを確認済み
- `WhiteBrowserDefaultGrid` / `Small` / `Big` は実 fixture 統合テストで、default `onUpdate` fallback と `onCreateThum` 描画、leave clear を確認済み
- `TutorialCallbackGrid -> WhiteBrowserDefaultList` の fixture 切替も実 host 統合テストで確認済み
- `TutorialCallbackGrid` の同一 fixture 再 navigate も実 host 統合テストで確認済み
  - `focus false -> select false -> clear -> leave` の順を probe で捕捉
- MainWindow UI 統合テストでも、`TutorialCallbackGrid` の DB切替後再描画と `TutorialCallbackGrid -> WhiteBrowserDefaultList` 切替を、実 fixture の DOM まで確認済み
- MainWindow UI 統合テストでも、`TutorialCallbackGrid` の `find / sort / addFilter` 再更新を、実 fixture + 実 MainDB の DOM まで確認済み
- MainWindow UI 統合テストでも、`TutorialCallbackGrid` の minimal reload と `external -> built-in` を、実 fixture の host / DOM まで確認済み

## スキン別 使用連携コマンド一覧

### `SimpleGridWB`

- 対象: `skin/SimpleGridWB/SimpleGridWB.htm`
- 使用コマンド
  - `wb.update()`
  - `wb.find()`
  - `wb.focusThum()`
  - `wb.getSkinName()`
  - `wb.getDBName()`
  - `wb.trace()`
- 使用 callback
  - `onUpdate(payload)` グローバル関数
- 使用 config
  - `multi-select : 1`
  - `scroll-id : view`
- 位置づけ
  - 互換ランタイムの基準スキン
  - callback よりも `update/find/focus` の基本導線監視に向く
