# 攻略対象スキン一覧と使用連携コマンド 2026-04-11

## 目的

- 外部スキン互換を進める時に、どのスキンから攻略すると効率が良いかを固定する
- 各スキンが使っている `wb.*` 連携コマンドと callback を一覧化し、未実装や不足挙動を追いやすくする

## 対象スキン一覧

### リポジトリ同梱スキン

1. `skin/DefaultGridWB`
2. `skin/SimpleGridWB`

### テスト用 WhiteBrowser スキン

1. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/TutorialCallbackGrid`
2. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultBig`
3. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultGrid`
4. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultList`
5. `Tests/IndigoMovieManager.Tests/TestData/WhiteBrowserSkins/WhiteBrowserDefaultSmall`

## 攻略優先順

1. `TutorialCallbackGrid`
   - callback が最も濃く、`onSkinEnter`、`onUpdate`、`onCreateThum`、`onSetFocus`、`onSetSelect`、`scrollSetting` をまとめて確認できる
2. `WhiteBrowserDefaultList`
   - `scroll-id` を持つため、一覧描画とスクロール領域の意味論確認に効く
3. `WhiteBrowserDefaultGrid`
   - 最小の `onCreateThum` 系グリッド描画として横展開しやすい
4. `WhiteBrowserDefaultSmall`
   - `score` 表示を含むため、旧 field alias の確認に向く
5. `WhiteBrowserDefaultBig`
   - `id` と `score` の見せ方が異なるため、描画差の確認に向く
6. `SimpleGridWB`
   - 既存互換ランタイムでの安定動作確認用ベースラインとして常時監視する

## 次の一手

1. `TutorialCallbackGrid` を正本対象にして、callback 発火順と payload 互換を固定する
2. `WhiteBrowserDefaultList` と `WhiteBrowserDefaultGrid` / `Small` / `Big` の fixture 実読込確認を土台に、手動受け入れ確認へ進む
3. `MainWindow` fixture 視点では `DB切替` / `external -> external` / `external -> built-in` / minimal reload / `find / sort / addFilter` まで実 fixture の host / DOM で確認済みなので、次は手動受け入れ確認と大量件数対策へ寄せる
4. `SimpleGridWB` は常時回帰監視用として維持しつつ、大量件数対策の入口となる段階読込 baseline としても使う
5. callback 未実装の外部 skin でも、compat fallback により「まずサムネが見える」状態を優先する
