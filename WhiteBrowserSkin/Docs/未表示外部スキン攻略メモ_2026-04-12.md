# 未表示外部スキン攻略メモ_2026-04-12

## 目的

- 一旦、未表示の外部スキン 6 本を正本メモとして固定する
- 今後の攻略順と確認観点を迷わず追えるようにする
- ゴールは「まず動画サムネが見える状態を全件で通す」こと

## 2026-04-12 開始時の残件 6 本

1. `TagInputRelation`
2. `umiFindTreeEve`
3. `Alpha2`
4. `Chappy`
5. `DefaultSmallWB`
6. `Search_table`

## 基本方針

- まずはレイアウト完成より「一覧に動画サムネが出る」を優先する
- callback 未実装差分は compat fallback で吸収する
- skin 個別の DOM 前提は、必要最小限の補完で通す
- 1 本ずつ潰しつつ、共通原因は compat 側へ戻して横展開する

## 攻略順

1. `DefaultSmallWB`
   - 既存 `Default*` 系との近さが高く、最短で 1 本減らしやすい
2. `Search_table`
   - table/list 系の DOM 前提を確認しやすい
3. `TagInputRelation`
   - relation 系の特殊 DOM や callback 前提を拾う
4. `umiFindTreeEve`
   - tree 系の構造差分を確認する
5. `Alpha2`
   - 個別テーマ差分として詰める
6. `Chappy`
   - 最後にテーマ固有差分を整理する

## 各 skin で最初に見る点

### 1. 一覧の入口

- `wb.onSkinEnter`
- `wb.update`
- `wb.onUpdate`
- `wb.onCreateThum`

### 2. サムネ表示の入口

- `mv.thum`
- `img{id}`
- `#thum{id}`
- `data-record-key`
- `scroll-id`

### 3. 既定 fallback で吸えるか

- `onCreateThum` 未実装でも最小サムネ生成で通るか
- `onUpdateThum` 未実装でも既定 `src` 差し替えで通るか

### 4. skin 固有差分か

- `table`
- `tree`
- relation 表示
- タグ入力 UI
- 独自 class 名

## 進め方

1. 1 本ずつ表示可否を確認する
2. 表示不可なら、失敗点を次の 3 種で切る
   - callback 不足
   - DOM 前提差分
   - skin 固有 script 依存
3. 共通原因なら compat へ戻す
4. 個別原因なら skin 単位の補完メモを追記する

## 完了条件

- 残り 6 本すべてで一覧に動画サムネが出る
- 少なくとも初回表示で「空欄ではない」ことを確認する
- 個別 skin の高度な見た目崩れは次段に回してよい

## 補足

- 2026-04-12 時点で compat runtime には、`onCreateThum` 未実装向け最小サムネ生成 fallback と、`onUpdateThum` 未実装向け `src` 差し替え fallback が入っている
- 2026-04-12 時点で `Search_table` と `Alpha2` は、custom `onUpdateThum(id, src)` が旧WB前提のまま残っていることを確認した
- そのため compat 側では、custom `onUpdateThum` が空振りした時だけ慣例 DOM (`img{id}` / `#thum{id} img` / `data-record-key`) への既定差し替えを後段で試す方針に寄せた
- 2026-04-12 時点で `TagInputRelation` と `umiFindTreeEve` は `#view` を持たない拡張寄り HTML だったため、compat 側で `view` 不在時は最小一覧コンテナを自動生成できるようにした
- そのため、次の調査は「何も出ない理由」を callback 不足だけでなく DOM 前提差分として見る
