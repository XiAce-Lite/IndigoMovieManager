# 運用ガイド_開発者向け配布手順の時系列解説_2026-04-05

最終更新日: 2026-04-05

## 1. この文書の目的

この文書は、開発者が配布導線を時系列で理解できるように、

1. エンジン側ビルド
2. 本体 app ビルド
3. GitHub Actions で WiX installer を生成
4. GitHub Release へ publish

の順で整理した運用ガイドである。

前提:

- Public repo は `IndigoMovieManager`
- Private repo は `IndigoMovieEngine`
- Public repo は app に機能を追加し、配る責務に集中する
- Private repo は worker / engine / package の正本を持つ

## 2. 全体の流れ

```text
1. Private 側に変更がある時だけ、Private repo で engine / worker を build / publish / release
2. Public repo で Private 成果物を同期し、verify 済み app package を作る
3. Public repo でその app package を入力に WiX installer を作る
4. GitHub Release に ZIP と installer.exe を載せる
```

要点:

- 毎回必須なのは `2 -> 3 -> 4`
- `1` は Private 側に変更がある時だけ前段で発生する
- installer は app package の後段工程だけを担当する
- worker や engine package を installer 側で再 build しない
- provenance 正本は app package 内の `rescue-worker.lock.json` と `privateEnginePackages`

### 2.1 2 パターンで見る

#### パターンA: Private 側に変更がある時

1. Private repo で worker / package を build / publish / release
2. Public repo でその成果物を同期
3. Public repo で app package を作る
4. Public repo で WiX installer を作る
5. GitHub Release に `ZIP + installer.exe` を載せる

#### パターンB: Private 側に変更がない時

1. 既存の Private release asset をそのまま同期
2. Public repo で app package を作る
3. Public repo で WiX installer を作る
4. GitHub Release に `ZIP + installer.exe` を載せる

つまり、Private 側の再 build / 再 release は毎回必須ではない。

## 3. 1. エンジン側ビルド

対象 repo:

- `%USERPROFILE%\source\repos\IndigoMovieEngine`

この段は、Private 側に変更がある時だけ実行する。

主な入口:

- `scripts\build_private_engine.ps1`
- `scripts\publish_private_engine.ps1`
- `scripts\pack_private_engine_packages.ps1`
- `.github\workflows\private-engine-publish.yml`

ここで作るもの:

### 3.1 worker publish

主な出力:

- `artifacts\rescue-worker\publish\Release-win-x64\`

中身:

- `IndigoMovieManager.Thumbnail.RescueWorker.exe`
- `rescue-worker-artifact.json`
- worker 実行に必要な DLL 群

### 3.2 worker ZIP

主な出力:

- `artifacts\rescue-worker\IndigoMovieManager.Thumbnail.RescueWorker-<version>-win-x64-compat-<CompatibilityVersion>.zip`

### 3.3 engine packages

主な出力:

- `artifacts\private-engine-packages\Release\IndigoMovieEngine.Thumbnail.Contracts.<version>.nupkg`
- `artifacts\private-engine-packages\Release\IndigoMovieEngine.Thumbnail.Engine.<version>.nupkg`
- `artifacts\private-engine-packages\Release\IndigoMovieEngine.Thumbnail.FailureDb.<version>.nupkg`
- `artifacts\private-engine-packages\Release\private-engine-packages-manifest.json`

### 3.4 GitHub 側で公開されるもの

Private repo の `private-engine-publish` では、最終的に次が GitHub 側の正本になる。

- worker release asset または publish artifact
- `Contracts / Engine / FailureDb` package release asset または artifact

補足:

- Public repo はこの成果物を同期して使う
- Public repo は engine / worker の正本 build を持たない

## 4. 2. 本体 app ビルド

対象 repo:

- `%USERPROFILE%\source\repos\IndigoMovieManager`

主な入口:

- `scripts\sync_private_engine_worker_artifact.ps1`
- `scripts\sync_private_engine_packages.ps1`
- `scripts\create_github_release_package.ps1`
- `scripts\invoke_release.ps1`

### 4.1 先に Private 成果物を同期する

この段は毎回実行する。  
ただし同期元は 2 通りある。

- Private 側を更新した直後の新しい release / artifact
- 既に存在する Private release asset

同期先:

- worker publish:
  - `artifacts\rescue-worker\publish\Release-win-x64\`
- engine packages:
  - `artifacts\private-engine-packages\Release\`

補助 metadata:

- `artifacts\rescue-worker\publish\Release-win-x64\rescue-worker-sync-source.json`
- `artifacts\private-engine-packages\Release\private-engine-packages-source.json`

### 4.2 verify 済み app package を作る

`scripts\create_github_release_package.ps1` が作るもの:

- publish 展開:
  - `artifacts\github-release\publish\<VersionLabel>-win-x64\`
- verify 済み package:
  - `artifacts\github-release\package\IndigoMovieManager-<VersionLabel>-win-x64\`
- 利用者向け ZIP:
  - `artifacts\github-release\IndigoMovieManager-<VersionLabel>-win-x64.zip`
- Release 本文 preview:
  - `artifacts\github-release\release-worker-lock-summary-<VersionLabel>-win-x64.md`

verify 済み package の中に含まれる重要物:

- `IndigoMovieManager.exe`
- `rescue-worker\*`
- `rescue-worker.lock.json`
- `rescue-worker-lock-summary.txt`
- `privateEnginePackages` を含む lock 情報

意味:

- この package が Public 側の正本配布素材
- WiX installer はこの package を唯一入力にする

## 5. 3. GitHub Actions で WiX installer を生成

対象 workflow:

- `.github\workflows\github-release-package.yml`

主な step:

1. Private worker / package を同期
2. `scripts\create_github_release_package.ps1` で verify 済み app package を作る
3. `scripts\create_wix_installer_from_release_package.ps1` で WiX installer を作る
4. Actions artifact を upload
5. tag 時だけ GitHub Release へ publish

### 5.1 WiX 生成時のローカル出力

workflow runner 内での主な出力:

- `artifacts\github-release\installer\<VersionLabel>-win-x64\msi\IndigoMovieManager.msi`
- `artifacts\github-release\installer\<VersionLabel>-win-x64\bundle\IndigoMovieManager.Bundle.exe`
- `artifacts\github-release\installer\IndigoMovieManager-Setup-<VersionLabel>-win-x64.exe`

意味:

- `MSI` は内部 package
- `Bundle.exe` は WiX の生出力
- 最終的に使うのは `Setup-*.exe`

### 5.2 Actions artifact 名

preview / tag 共通で upload されるもの:

- `github-release-package`
  - `artifacts\github-release\*.zip`
- `github-release-installer`
  - `artifacts\github-release\installer\*.exe`
- `github-release-body-preview`
  - `artifacts\github-release\release-worker-lock-summary-*.md`

補足:

- `MSI`
- `bundle\IndigoMovieManager.Bundle.exe`
- `*.wixpdb`

は artifact としては上げない

## 6. 4. GitHub Release へ publish

tag push 時だけ、`softprops/action-gh-release@v2` が GitHub Release へ asset を載せる。

GitHub Release へ publish されるもの:

- `IndigoMovieManager-<version>-win-x64.zip`
- `IndigoMovieManager-Setup-<version>-win-x64.exe`

GitHub Release へ publish されないもの:

- `IndigoMovieManager.msi`
- `IndigoMovieManager.Bundle.exe`
- `*.wixpdb`
- `release-worker-lock-summary-*.md`
- `package\IndigoMovieManager-<VersionLabel>-win-x64\` ディレクトリそのもの
